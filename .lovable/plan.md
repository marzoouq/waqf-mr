
# خطة شاملة محدّثة: أرشفة السنوات المالية + إدارة الفواتير + إصلاح البيانات

## تقييم التطابق مع المعايير المحلية والدولية

### المعايير المرجعية:
- **SOCPA** (الهيئة السعودية للمحاسبين): معايير المحاسبة السعودية
- **IFRS**: المعايير الدولية للتقارير المالية
- **ZATCA**: هيئة الزكاة والضريبة والجمارك
- **نظام الأوقاف السعودي**: الهيئة العامة للأوقاف
- **ISA**: معايير التدقيق الدولية

### ما يتوافق حالياً (لا يحتاج تعديل):
1. التسلسل المالي الهرمي (Income -> Expenses -> VAT -> Zakat -> Shares -> Revenue) - متوافق مع IFRS/SOCPA
2. فصل حصص الناظر (10%) والواقف (5%) من أساس موحد - متوافق مع أحكام الأوقاف الشرعية
3. التوزيع على المستفيدين بنسبة 2:1 (ذكر:أنثى) - متوافق مع الميراث الشرعي
4. حماية البيانات الشخصية (Data Masking) - متوافق مع نظام حماية البيانات الشخصية السعودي
5. سياسات RLS للفصل بين الصلاحيات - متوافق مع معايير الأمان

### الفجوات المكتشفة والحلول:

| # | المعيار | الفجوة | الحل في هذه الخطة |
|---|---------|--------|-------------------|
| 1 | SOCPA - الفترات المالية | السنة المالية نص يدوي بدون تواريخ منظمة | جدول fiscal_years بتواريخ بداية/نهاية |
| 2 | IFRS - مبدأ الاستمرارية | لا توجد آلية ترحيل أرصدة | ترحيل تلقائي عند إقفال السنة |
| 3 | ZATCA - التوثيق | فواتير منفصلة عن المصروفات | ربط كل مصروف بمستند إثبات |
| 4 | نظام الأوقاف | لا يوجد قفل بعد الإقفال | منع التعديل على سنة مقفلة |
| 5 | ISA - سجل المراجعة | لا يوجد Audit Trail | سجل تلقائي لكل تعديل/حذف |
| 6 | بيانات غير متسقة | 17 عقد منتهي مسجل كنشط + خطأ تقريب الحصص | إصلاح فوري + آلية تحديث تلقائي |

## المرحلة 1: إصلاحات البيانات الفورية

### 1.1 تحديث العقود المنتهية (17 عقد)
- تحديث جميع العقود التي انتهت تواريخها من status = 'active' إلى status = 'expired'
- إنشاء Edge Function (auto-expire-contracts) تعمل يومياً لتحديث العقود المنتهية تلقائياً

### 1.2 تصحيح نسب حصص المستفيدين
- المجموع الحالي: 100.000003% (خطأ تقريب 0.000003%)
- الحل: تعديل حصة آخر مستفيد لاستيعاب فرق التقريب ليصبح المجموع 100.000000% بالضبط

## المرحلة 2: نظام أرشفة السنوات المالية

### 2.1 جدول fiscal_years الجديد
يحتوي على:
- معرف فريد (id)
- تسمية السنة (label) مثل "2024-2025"
- تاريخ البداية (start_date) وتاريخ النهاية (end_date)
- الحالة (status): active / closed
- الطابع الزمني (created_at)

### 2.2 تعديل الجداول الحالية
- إضافة عمود fiscal_year_id في جداول: income, expenses, invoices
- ربط البيانات الحالية تلقائياً بالسنة المالية 2024-2025 (حسب التاريخ)
- إنشاء سنة مالية جديدة 2025-2026 كسنة نشطة

### 2.3 واجهة اختيار السنة المالية
- مكوّن FiscalYearSelector يظهر في أعلى صفحات: الدخل، المصروفات، الفواتير، الحسابات
- السنة النشطة تُعرض افتراضياً
- إمكانية عرض "جميع السنوات" أو سنة محددة
- البيانات الجديدة تُربط تلقائياً بالسنة النشطة

### 2.4 آلية إقفال السنة المالية
- زر "إقفال السنة" في صفحة الحسابات الختامية
- عند الإقفال:
  - تتحول السنة من active إلى closed
  - يُحسب الرصيد المتبقي ويُرحل كـ waqf_corpus_previous للسنة الجديدة
  - تُنشأ سنة مالية جديدة تلقائياً
  - يُمنع التعديل على بيانات السنة المقفلة (متوافق مع نظام الأوقاف)

---

## المرحلة 3: ربط المصروفات بالفواتير (التوثيق الورقي)

### 3.1 تحسين نموذج إضافة المصروف
- إضافة زر "إرفاق فاتورة" داخل نموذج المصروف مباشرة
- عند الحفظ يُنشأ سجل في جدول invoices مرتبط بالمصروف (expense_id)
- دعم إرفاق عدة ملفات (صور أو PDF) لنفس المصروف

### 3.2 تصميم عرض المصروفات المحسّن
- عمود "المرفقات" جديد في جدول المصروفات يعرض عدد الفواتير بأيقونة
- صفوف قابلة للتوسع (Expandable Rows) لعرض الفواتير المرفقة عند النقر
- بطاقات ملخص محسّنة: إجمالي المصروفات + نسبة التوثيق (كم مصروف له فاتورة)

### 3.3 صفحة الفواتير المحسّنة
- إضافة عرض شبكي (Grid View) بجانب العرض الجدولي الحالي
- العرض الشبكي يعرض صورة مصغرة للفاتورة مع المبلغ والنوع
- فلترة حسب: النوع، العقار، السنة المالية، الحالة
- نفس التحسينات تنعكس على واجهة المستفيد (InvoicesViewPage)

---

## المرحلة 4: سجل المراجعة (Audit Trail) - متوافق مع ISA

### 4.1 جدول audit_log الجديد
يسجل تلقائياً كل عملية تعديل أو حذف على الجداول المالية (income, expenses, accounts, distributions) عبر Database Trigger:
- نوع العملية (INSERT/UPDATE/DELETE)
- الجدول المتأثر
- البيانات القديمة والجديدة (JSONB)
- المستخدم الذي نفذ العملية
- الطابع الزمني

هذا الجدول للقراءة فقط (لا يمكن حذف سجلاته) لضمان النزاهة المالية.
