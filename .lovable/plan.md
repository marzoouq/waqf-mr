

# تقرير الفحص الأمني الشامل — مستوعبات التخزين وسياسات RLS

## ملخص تنفيذي

تم فحص **37 جدول** (جميعها مفعّل عليها RLS)، و**134 سياسة RLS**، و**2 مستوعب تخزين** بإجمالي **13 سياسة تخزين**، و**11 Edge Function**. النظام محصّن بشكل ممتاز مع **ثغرة واحدة نشطة** تحتاج إصلاح فوري.

---

## النتائج

### خطير — يحتاج إصلاح فوري

| # | الثغرة | التفاصيل |
|---|--------|----------|
| 1 | **سياسة تخزين فواتير متساهلة** | سياسة `Authenticated users can view invoices` لا تزال موجودة على مستوعب `invoices` الخاص، تسمح لأي مستخدم مسجل (بما فيهم المستفيد والواقف) بتحميل ملفات PDF الفواتير. هذا يتعارض مع سياسات RLS على جداول `payment_invoices` و`invoices` التي تقيّد الوصول بالناظر والمحاسب. **ملاحظة:** سجل الذاكرة يشير إلى أن هذه السياسة "حُذفت" لكنها لا تزال موجودة فعلياً في قاعدة البيانات. |

### مقبول — مخاطر مدروسة ومُوثّقة

| # | البند | الحالة |
|---|-------|--------|
| 2 | مستوعب `waqf-assets` عام | مقصود — شعارات وخطوط فقط، الكتابة مقيّدة بالناظر |
| 3 | عرضا `beneficiaries_safe` و`contracts_safe` بنمط Security Definer | مقصود — مع `security_barrier=true` وتقنيع PII عبر `CASE WHEN` |
| 4 | Realtime بدون تفويض قنوات | لا يُستخدم Broadcast/Presence — فقط `postgres_changes` المحمي بـ RLS |
| 5 | `pgcrypto` في schema الإضافات | مُثبّت في `extensions` وليس `public` — إنذار كاذب |
| 6 | مفاتيح ZATCA الخاصة | مشفّرة عبر trigger + الوصول للناظر فقط |

---

## خطة الإصلاح

### حذف سياسة التخزين المتساهلة

Migration واحدة لحذف السياسة:

```sql
DROP POLICY IF EXISTS "Authenticated users can view invoices" 
  ON storage.objects;
```

السياسات المتبقية (`Admins can read invoices` + `Accountants can read invoices` + `Admin and accountant can view invoices`) تغطي الوصول المطلوب بالكامل.

### الملفات المتأثرة

| الملف | التغيير |
|-------|---------|
| `supabase/migrations/xxx_drop_invoice_storage_open_policy.sql` | Migration لحذف السياسة |
