

# خطة إدارة العقود المنتهية وتنبيهات التجديد

## الوضع الحالي

- جميع العقود الـ20 في قاعدة البيانات بحالة **"منتهي"** (expired)
- يوجد زر **"تجديد"** يدوي لكل عقد في صفحة العقود (ينشئ عقداً جديداً بالبيانات نفسها)
- يوجد Edge Function لتنبيه العقود القريبة من الانتهاء (`check-contract-expiry`) ووظيفة أخرى لتحديث الحالة تلقائياً (`auto-expire-contracts`)
- لا توجد آلية **تجديد جماعي** أو **تنبيه تلقائي بعد الانتهاء الفعلي**

## ما سيتم تنفيذه

### 1. زر تجديد جماعي للعقود المنتهية

إضافة زر **"تجديد جميع العقود المنتهية"** في صفحة العقود يظهر عندما توجد عقود منتهية، يقوم بـ:
- عرض مربع تأكيد يوضح عدد العقود التي ستُجدد
- إنشاء عقود جديدة بنفس بيانات العقود المنتهية (المستأجر، العقار، الوحدة، قيمة الإيجار، نوع الدفع)
- تحديث تواريخ البداية والنهاية تلقائياً لسنة جديدة
- ربط العقود الجديدة بالسنة المالية النشطة
- ترقيم العقود الجديدة تلقائياً (مثلاً: C-001 تصبح C-001-R1)

### 2. تحسين تنبيهات الناظر

تعديل وظيفة `check-contract-expiry` لتشمل:
- تنبيه إضافي عن العقود **المنتهية فعلياً** التي لم تُجدد بعد (وليس فقط القريبة من الانتهاء)
- رسالة واضحة: "يوجد X عقد منتهي لم يتم تجديده"

### 3. شريط تنبيه في لوحة التحكم

إضافة شريط تحذيري في أعلى صفحة العقود عند وجود عقود منتهية، مع رابط مباشر لزر التجديد الجماعي.

## التفاصيل التقنية

### تعديل `ContractsPage.tsx`
- إضافة زر "تجديد جماعي" مع `AlertDialog` للتأكيد
- منطق التجديد: لكل عقد منتهٍ، إنشاء عقد جديد بنفس البيانات مع تواريخ جديدة (من اليوم + سنة)
- إضافة شريط تنبيه `Alert` عند وجود عقود منتهية
- إرسال إشعار للناظر والمستفيدين عند التجديد الجماعي

### تعديل `check-contract-expiry/index.ts`
- إضافة استعلام للعقود المنتهية (`status = 'expired'`) وإرسال تنبيه تذكيري أسبوعي للناظر بعدد العقود غير المجددة
- الحفاظ على التنبيه الحالي للعقود القريبة من الانتهاء (30 يوماً)

### لا تغييرات على قاعدة البيانات
- الهيكل الحالي يدعم كل المطلوب (حقول status، payment_type، fiscal_year_id موجودة)
