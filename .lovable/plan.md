

# إضافة أمر تقارير التغطية + تقييم ثغرة ajv

## المهمة 1: إضافة سكريبت تقارير التغطية

الإعداد موجود بالفعل في `vitest.config.ts` (مزود v8، مجلدات التغطية محددة). المطلوب فقط إضافة سكريبت مختصر في `package.json`:

```json
"test:coverage": "vitest run --coverage"
```

**الملف المتأثر:** `package.json` فقط (إضافة سطر واحد في scripts)

بعد الإضافة سيتم تشغيل الأمر لعرض نتائج التغطية الفعلية.

## المهمة 2: تقييم ثغرة ajv (ReDoS)

**النتيجة: لا خطر فعلي.**

| الحزمة | الإصدار | الاستخدام | التقييم |
|---|---|---|---|
| `ajv` 8.18.0 | أحدث من 8.17.1 (الإصدار المصاب) | تبعية غير مباشرة لـ `workbox-build` (build-time فقط) | آمن |
| `ajv` 6.12.6 | إصدار مختلف (v6) | تبعية dev لـ `eslint` (لا تعمل في الإنتاج) | آمن |

- الثغرة تؤثر على ajv حتى الإصدار 8.17.1 — المشروع يستخدم 8.18.0 (مُصلح)
- كلا الإصدارين تبعيات تطوير فقط ولا تصل للمتصفح أو الخادم
- لا يوجد استخدام مباشر لـ ajv أو `$data: true` في كود المشروع

لا حاجة لأي إجراء بخصوص هذه الثغرة.
