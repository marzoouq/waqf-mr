

# خطة إصلاح فشل النشر النهائية

## المشكلة المؤكدة بالدليل

| الملف | إصدار next-themes | الحالة |
|-------|-------------------|--------|
| `package.json` | `^0.4.6` | محدّث ✅ |
| `bun.lock` | `0.4.6` | محدّث ✅ |
| `package-lock.json` | `0.3.0` | **قديم ❌** |

GitHub Actions يستخدم `npm ci` الذي يقرأ `package-lock.json` حصرياً — فيُثبّت `0.3.0` → تعارض مع React 19 → فشل البناء.

## خطة التنفيذ

### الخطوة 1: إعادة توليد `package-lock.json`
- حذف الملف الحالي وإعادة إنشائه بحيث يتضمن `next-themes@0.4.6`
- هذا يحل مشكلة الـ 811 تشغيل فاشل في GitHub Actions نهائياً

### الخطوة 2: فشل النشر (restore point)
- هذا خطأ منصة وليس كود — إعادة النشر بعد تطبيق الخطوة 1 ستعمل تلقائياً لأن المنصة تعيد المحاولة
- لا يوجد إجراء كود مطلوب لهذه المشكلة

### التفاصيل التقنية
- الملف الوحيد المتأثر: `package-lock.json`
- السبب: عند ترقية `next-themes` في `package.json` تم تحديث `bun.lock` تلقائياً لكن `package-lock.json` لم يُحدَّث
- `.github/workflows/test.yml` يحتوي بالفعل `--legacy-peer-deps` كشبكة أمان إضافية
