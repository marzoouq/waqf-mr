
# خطة تحسين آلية إقفال السنة المالية وأرشفة البيانات

## الوضع الحالي

عملية الإقفال الحالية تقوم بـ:
1. تغيير حالة السنة المالية إلى "مغلقة"
2. إنشاء سنة مالية جديدة
3. ترحيل الرصيد المتبقي (`remainingBalance`) كـ `waqf_corpus_previous` للسنة الجديدة

لكن هناك مشاكل:
- لا يتم حفظ حساب ختامي نهائي تلقائياً قبل الإقفال (يعتمد على ضغط الناظر يدوياً)
- سجل الحساب الجديد يُنشأ بدون ربط بـ `fiscal_year_id`
- لا يوجد تأكيد أن الحساب الختامي الحالي محفوظ قبل الإقفال

## خطة الإصلاح

### 1. تحسين عملية الإقفال في `AccountsPage.tsx`

عند الضغط على "تأكيد الإقفال":

**الخطوة أ:** حفظ الحساب الختامي النهائي تلقائياً للسنة الحالية (بنفس منطق `handleCreateAccount`) لضمان أرشفة كافة الأرقام.

**الخطوة ب:** إقفال السنة المالية (تغيير الحالة إلى `closed`).

**الخطوة ج:** إنشاء السنة المالية الجديدة مع ترحيل `remainingBalance` فقط كـ `waqf_corpus_previous`.

**الخطوة د:** إرسال إشعار للمستفيدين بإقفال السنة.

### 2. تحسين رسالة التأكيد

إضافة ملخص واضح يوضح:
- المبلغ الذي سيتم ترحيله (الرصيد المتبقي)
- أن جميع البيانات ستبقى محفوظة للاطلاع عليها
- أنه لن يمكن التعديل بعد الإقفال

### 3. تحسين تقني

في `handleCloseYear`:
- حفظ الحساب الختامي أولاً قبل الإقفال
- إبطال كاش `accounts` بعد العملية
- إضافة إشعار للمستفيدين

## التفاصيل التقنية

### الملف: `src/pages/dashboard/AccountsPage.tsx`

تعديل دالة `handleCloseYear` لتشمل:

```text
1. حفظ حساب ختامي نهائي (createAccount) بكافة الأرقام الحالية
2. إقفال السنة (update status = 'closed')
3. إنشاء سنة جديدة
4. إنشاء سجل حساب للسنة الجديدة مع waqf_corpus_previous = remainingBalance فقط
5. إبطال كاش: fiscal_years, accounts, income, expenses
6. إشعار المستفيدين
```

تحديث نص `AlertDialogDescription` ليعرض ملخصاً أوضح:
- "سيتم حفظ الحساب الختامي النهائي وأرشفة جميع البيانات"
- "سيتم ترحيل المبلغ المتبقي (X ر.س) كرقبة وقف للسنة الجديدة"
