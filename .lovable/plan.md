

# خطة إضافة لائحة تنظيم أعمال الوقف والنظارة

## ملخص المشروع
إضافة صفحة احترافية للائحة التنفيذية لتنظيم أعمال الوقف والنظارة، مع تخزين محتوى اللائحة في قاعدة البيانات لتمكين الناظر من تعديلها، وإتاحة القراءة للمستفيدين والواقف.

---

## نتائج فحص اللائحة (المنهج الجنائي)

### التكرارات المكتشفة والمحذوفة:
1. **"حماية الاصول"** مكررة في الجزء السابع (الفصل التاسع) والجزء الثالث عشر (الفصل السابع عشر) -- يتم دمجها في موضع واحد
2. **"تنمية الوقف"** مكررة في الفصل العاشر والفصل السابع عشر -- يتم دمجها
3. **"تدريب العاملين"** مكرر في الجزء الثامن والجزء الرابع عشر -- يتم دمجه
4. **"توثيق وارشفة البيانات"** مكرر في الجزء التاسع والجزء الخامس عشر -- يتم دمجه
5. **"خطط الاستثمار"** مكررة جزئيا بين الفصل الثامن والفصل الخامس عشر -- يتم دمجها

### الاضافات المقترحة (نقص مكتشف):
1. **مادة "الاحكام الانتقالية"**: تحديد فترة سريان اللائحة ومراجعتها الدورية
2. **مادة "ال توزيعات"**: تفصيل آلية حساب وتوزيع الحصص على المستفيدين (85% ريع الوقف)
3. **مادة "السرية وحماية المعلومات"**: ضوابط حماية بيانات المستفيدين الشخصية

## طريقة الاضافة الاحترافية

### النهج: تخزين اللائحة في قاعدة البيانات كمحتوى قابل للتعديل

بدلا من كتابة اللائحة كنص ثابت في الكود (مثل صفحة الخصوصية)، سيتم تخزينها في جدول `waqf_bylaws` في قاعدة البيانات. هذا يتيح للناظر تعديل المحتوى دون الحاجة لتعديل الكود.

---

## التفاصيل التقنية

### 1. جدول قاعدة البيانات الجديد: `waqf_bylaws`

```text
waqf_bylaws
  id: uuid (PK)
  part_number: integer        -- رقم الجزء
  chapter_number: integer     -- رقم الفصل (nullable)
  part_title: text             -- عنوان الجزء
  chapter_title: text          -- عنوان الفصل (nullable)
  content: text                -- المحتوى (يدعم Markdown)
  sort_order: integer          -- ترتيب العرض
  is_visible: boolean          -- اظهار/اخفاء
  created_at: timestamptz
  updated_at: timestamptz
```

**سياسات RLS:**
- الناظر: صلاحيات كاملة (ALL)
- المستفيدون والواقف: قراءة فقط (SELECT)

### 2. الملفات الجديدة

| الملف | الوصف |
|:---|:---|
| `src/pages/dashboard/BylawsPage.tsx` | صفحة الناظر: عرض + تعديل اللائحة |
| `src/pages/beneficiary/BylawsViewPage.tsx` | صفحة المستفيد: عرض فقط |
| `src/hooks/useBylaws.ts` | Hook لجلب وتعديل بنود اللائحة |

### 3. التعديلات على الملفات الموجودة

| الملف | التعديل |
|:---|:---|
| `src/App.tsx` | اضافة مسارين: `/dashboard/bylaws` و `/beneficiary/bylaws` |
| `src/components/DashboardLayout.tsx` | اضافة رابط "اللائحة التنظيمية" في القائمة الجانبية للناظر والمستفيد |

### 4. واجهة الناظر (`BylawsPage.tsx`)

- عرض اللائحة كاملة مقسمة حسب الاجزاء والفصول باستخدام Accordion
- زر تعديل لكل بند يفتح Dialog مع محرر نصي (Textarea)
- امكانية اخفاء/اظهار بنود محددة
- زر طباعة/تصدير PDF
- يدعم عرض النص بتنسيق Markdown (react-markdown مثبتة بالفعل)

### 5. واجهة المستفيد (`BylawsViewPage.tsx`)

- عرض اللائحة بنفس التنسيق لكن بدون ازرار التعديل
- تنقل سلس بين الاجزاء عبر فهرس جانبي
- تصميم متجاوب للجوال

### 6. بيانات اللائحة الاولية

سيتم ادراج محتوى اللائحة المنقح (بعد حذف التكرارات واضافة النواقص) كبيانات اولية عبر migration SQL.

### 7. التكامل مع سجل المراجعة

سيتم اضافة trigger لسجل المراجعة (audit_log) على جدول `waqf_bylaws` لتتبع اي تعديل يجريه الناظر على بنود اللائحة.

## ملخص التغييرات

- **1 جدول جديد** في قاعدة البيانات مع RLS وtrigger للمراجعة
- **3 ملفات جديدة** (صفحتان + hook)
- **2 ملف معدل** (App.tsx + DashboardLayout.tsx)
- **محتوى اللائحة** منقح ومنظم (حذف 5 تكرارات + اضافة 3 بنود ناقصة)
