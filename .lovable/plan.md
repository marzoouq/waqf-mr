

# إضافة دور "المحاسب" (Accountant) إلى نظام إدارة الوقف

## ملخص
إضافة دور رابع "محاسب" يمتلك صلاحيات مشابهة للناظر في الجوانب المالية والتشغيلية، مع استثناء إدارة المستخدمين والإعدادات العامة وسجل المراجعة.

## صلاحيات المحاسب

| القسم | عرض | إضافة/تعديل | حذف |
|---|---|---|---|
| العقارات | نعم | نعم | نعم |
| العقود | نعم | نعم | نعم |
| الدخل | نعم | نعم | نعم |
| المصروفات | نعم | نعم | نعم |
| الفواتير | نعم | نعم | نعم |
| الحسابات الختامية | نعم | نعم | نعم |
| المستفيدين | نعم | نعم | نعم |
| التقارير | نعم | تصدير | - |
| اللائحة التنظيمية | نعم | لا | لا |
| المراسلات | نعم | نعم | لا |
| إدارة المستخدمين | لا | لا | لا |
| الإعدادات | لا | لا | لا |
| سجل المراجعة | عرض فقط | لا | لا |

## خطوات التنفيذ

### الجزء 1: قاعدة البيانات (Migration)

1. إضافة قيمة `accountant` إلى نوع `app_role` الموجود:
```sql
ALTER TYPE public.app_role ADD VALUE 'accountant';
```

2. تحديث سياسات RLS لجميع الجداول المالية للسماح للمحاسب بالعمليات الكاملة (CRUD):
   - `properties`, `contracts`, `income`, `expenses`, `invoices`, `accounts`, `beneficiaries`, `units`, `tenant_payments`, `distributions`
   - إضافة سياسة `Accountants can manage [table]` لكل جدول

3. تحديث سياسات العرض فقط:
   - `audit_log`: إضافة المحاسب لسياسة العرض
   - `waqf_bylaws`: إضافة المحاسب لسياسة العرض
   - `fiscal_years`: إضافة المحاسب لسياسات العرض والإدارة
   - `app_settings`: إضافة المحاسب لسياسة القراءة
   - `notifications`, `conversations`, `messages`: تعمل تلقائياً عبر السياسات الحالية

4. تحديث دالة `prevent_closed_fiscal_year_modification` للسماح للمحاسب بالتعديل مثل الناظر

### الجزء 2: الكود (Frontend)

1. **`src/types/database.ts`**: إضافة `'accountant'` إلى نوع `AppRole`

2. **`src/constants/index.ts`**: إضافة تسمية `accountant: 'محاسب'` في `ROLE_LABELS`

3. **`src/components/DashboardLayout.tsx`**:
   - إضافة قائمة روابط خاصة بالمحاسب (مثل الناظر بدون إدارة المستخدمين والإعدادات)
   - تعديل شرط `role` لدعم `'accountant'`

4. **`src/App.tsx`**:
   - إضافة `'accountant'` إلى `allowedRoles` في المسارات المالية والتشغيلية
   - استثناء مسارات: `/dashboard/users`, `/dashboard/settings`
   - السماح بعرض سجل المراجعة: `/dashboard/audit-log`

5. **`src/components/ProtectedRoute.tsx`**: إضافة `'accountant'` إلى نوع `allowedRoles`

6. **`src/contexts/AuthContext.tsx`**: لا يحتاج تعديل (يقرأ الدور من `user_roles` تلقائياً)

7. **`src/pages/Auth.tsx`**: تعديل التوجيه بعد تسجيل الدخول ليوجه المحاسب إلى `/dashboard`

8. **Edge Functions** (`admin-manage-users`, `guard-signup`): تحديث فحص الصلاحيات للسماح للمحاسب حيث يلزم

### الجزء 3: إنشاء حساب المحاسب

- يتم إنشاء حسابات المحاسبين عبر صفحة إدارة المستخدمين (بواسطة الناظر فقط)
- تعديل صفحة `UserManagementPage` لإضافة خيار دور "محاسب" في قائمة الأدوار المتاحة

## ملاحظات أمنية
- المحاسب لا يستطيع تعديل الأدوار أو إنشاء مستخدمين جدد
- المحاسب لا يستطيع تعديل إعدادات النظام (النسب المالية، اللائحة، إلخ)
- جميع عمليات المحاسب مسجلة في سجل المراجعة (Audit Log) تلقائياً عبر المشغلات الموجودة
- المحاسب يمكنه عرض سجل المراجعة لكن لا يمكنه حذفه أو تعديله
