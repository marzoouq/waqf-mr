
# إضافة تحقق من بريد المستفيدين في صفحة إدارة المستخدمين

## الهدف

إضافة تنبيه تلقائي في صفحة إدارة المستخدمين يكشف المستفيدين الذين ليس لديهم بريد إلكتروني مسجل في جدول `beneficiaries`، مما يمنع مشكلة "رقم الهوية غير مسجل" مستقبلاً.

## التغييرات

### تعديل `src/pages/dashboard/UserManagementPage.tsx`

1. **إضافة استعلام جديد** يجلب المستفيدين الذين `email` لديهم `NULL` أو فارغ، أو الذين `user_id` لديهم `NULL`:

```text
const { data: orphanedBeneficiaries = [] } = useQuery({
  queryKey: ['orphaned-beneficiaries'],
  queryFn: async () => {
    const { data } = await supabase
      .from('beneficiaries')
      .select('id, name, email, user_id')
      .or('email.is.null,email.eq.,user_id.is.null');
    return data || [];
  },
});
```

2. **إضافة بطاقة تنبيه** تظهر أعلى جدول المستخدمين عند وجود مستفيدين بدون بريد أو بدون ربط بحساب مستخدم. تعرض:
   - عدد المستفيدين المتأثرين
   - أسماءهم
   - توضيح أن تسجيل الدخول بالهوية لن يعمل لهم
   - تُستخدم مكونات `Alert` و `AlertTriangle` الموجودة أصلاً

3. **إضافة علامة تحذير في جدول المستخدمين** بجانب المستخدمين من نوع `beneficiary` الذين ليس لديهم سجل مطابق في جدول المستفيدين (مقارنة عبر البريد).

## ملاحظات
- البيانات الحالية سليمة: جميع الـ 14 مستفيداً لديهم بريد و `user_id`
- هذا التحقق وقائي لمنع المشكلة مستقبلاً عند إضافة مستفيدين جدد
- لا تغيير في قاعدة البيانات أو Edge Functions
