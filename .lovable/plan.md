## 2. الفجوات المكتشفة بالتفصيل

### فجوة حرجة 1: اللائحة التنظيمية مفقودة بالكامل في الإنتاج

- **التطوير**: 20 بنداً كاملاً (المقدمة + 19 فصلاً)
- **الإنتاج**: 0 بنود - الجدول فارغ تماماً
- **الأثر المرئي**: صفحة "اللائحة التنظيمية" فارغة تماماً في الموقع المنشور
- **السبب الجذري**: البيانات لا تُنقل عند النشر - فقط هيكل الجداول (Schema) ينتقل تلقائياً، أما الصفوف فتبقى محصورة في بيئتها

**الإصلاح**: يجب تشغيل 20 عبارة INSERT في بيئة الإنتاج عبر Cloud View > Run SQL (Live). سيتم توليد الاستعلامات الكاملة عند الموافقة.

## 3. لماذا تحدث هذه الفجوات؟ (السبب الجذري)

**آلية النشر (Publish) في Lovable Cloud:**
- عند النشر، يُنقل **الكود + هيكل قاعدة البيانات (Schema)** فقط
- **البيانات (الصفوف)** لا تُنقل أبداً -- كل بيئة لها بياناتها المستقلة
- **ملفات التخزين** لا تُنقل -- كل بيئة لها مخزنها المستقل
- هذا التصميم **مقصود** لحماية بيانات الإنتاج الحقيقية من العبث أثناء التطوير

**الفجوة تحدث عندما:**
1. تُضاف بيانات في التطوير (مثل اللائحة) ولا تُنقل يدوياً للإنتاج
2. تُطبّق تصحيحات في التطوير (مثل الحسابات) ولا تُكرّر في الإنتاج
3. تُرفع ملفات في التطوير (مثل الشعار) ولا تُرفع في الإنتاج

## 4. خطة الإصلاح الشاملة

### الخطوة 1: نقل بيانات اللائحة (20 بنداً)
سيتم توليد استعلامات INSERT كاملة تحتوي نصوص الـ20 فصلاً بالكامل لتشغيلها في الإنتاج.

### الخطوة 2: تصحيح الحسابات الختامية
```text
UPDATE accounts SET
  total_expenses = 121722.10,
  admin_share = 112876.11,
  waqif_share = 56438.05,
  net_after_expenses = 1368657.90,
  net_after_vat = 1275744.97,
  waqf_revenue = 1102913.98,
  distributions_amount = 995000.03
WHERE fiscal_year = '2024-2025';
```

### الخطوة 3: إضافة الإعدادات المفقودة
```text
INSERT INTO app_settings (key, value) VALUES
  ('bylaws_published', 'false'),
  ('beta_banner_settings', '{"enabled":true,"text":"نسخة تجريبية (Beta) — يجري تطوير النظام وتحسينه، ونرحّب بملاحظاتكم لدعم جودة الخدمة","color":"green","position":"top","dismissible":true}')
ON CONFLICT (key) DO NOTHING;
```

### الخطوة 4: رفع الشعار وتحديث رابطه
1. رفع ملف logo.png إلى مخزن waqf-assets في بيئة الإنتاج (عبر Cloud View > Storage)
2. تحديث الإعداد:
```text
UPDATE app_settings 
SET value = '[رابط الشعار بعد الرفع]', updated_at = now()
WHERE key = 'waqf_logo_url';
```

## 5. ملخص التقييم النهائي

```text
البند                              | التقييم
-----------------------------------+------------------
هيكل قاعدة البيانات (Schema)       | متطابق 100%
البيانات المالية الخام (دخل/مصروف) | متطابقة 100%
الأدوار والصلاحيات (RLS + Roles)   | متطابقة 100%
بيانات المستفيدين والعقارات        | متطابقة 100%
الحسابات الختامية                  | انحراف تقريب (يحتاج تحديث)
اللائحة التنظيمية                  | مفقودة بالكامل في الإنتاج
الإعدادات                          | ناقص 2 من 22
الشعار                             | مفقود في الإنتاج
الكود البرمجي                      | متطابق (ينتقل تلقائياً عند النشر)
```

**الخلاصة**: الكود والهيكل سليمان ومتطابقان. المشكلة محصورة في **4 فجوات بيانات** تحتاج نقلاً يدوياً عبر Cloud View > Run SQL (Live). عند الموافقة، سيتم توليد جميع استعلامات SQL الجاهزة للتنفيذ.
