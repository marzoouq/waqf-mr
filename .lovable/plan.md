
# خطة: قفل التسجيل العام + تسجيل الدخول برقم الهوية أو الإيميل

## الهدف
1. منح الناظر صلاحية تفعيل/إيقاف التسجيل العام من لوحة التحكم (بحيث يتم التسجيل فقط من لوحة الناظر)
2. السماح بتسجيل الدخول باستخدام رقم الهوية الوطنية بالإضافة للبريد الإلكتروني

## التعديلات المطلوبة

### 1. جدول إعدادات النظام (Migration جديد)
- إنشاء جدول `app_settings` لتخزين إعداد قفل التسجيل
- يحتوي على: `key` (نص فريد) و `value` (نص)
- إدراج إعداد افتراضي: `registration_enabled = false` (مقفل افتراضيا)
- سياسات RLS: الناظر يقرأ ويعدل، والجميع يقرأ فقط

### 2. إضافة حقل رقم الهوية للمستفيدين (Migration)
- إضافة عمود `national_id` (نص، اختياري) لجدول `beneficiaries`
- يُستخدم كمعرف بديل لتسجيل الدخول

### 3. صفحة تسجيل الدخول (`Auth.tsx`)
- إزالة تبويب "حساب جديد" واستبداله بنموذج تسجيل دخول فقط (افتراضيا)
- جلب إعداد `registration_enabled` من قاعدة البيانات عند تحميل الصفحة
- إظهار تبويب التسجيل فقط إذا كان التسجيل مفعلا
- إضافة خيار تسجيل الدخول بـ "رقم الهوية" أو "البريد الإلكتروني":
  - المستخدم يختار طريقة الدخول
  - عند اختيار رقم الهوية: يبحث النظام عن المستفيد المرتبط بهذا الرقم ويستخدم بريده الإلكتروني لتسجيل الدخول

### 4. لوحة تحكم الناظر - إعدادات التسجيل (`UserManagementPage.tsx`)
- إضافة مفتاح تبديل (Switch) في أعلى الصفحة لتفعيل/إيقاف التسجيل العام
- عند الإيقاف: لا يظهر تبويب "حساب جديد" في صفحة تسجيل الدخول
- إضافة حقل "رقم الهوية" في نموذج إنشاء المستخدم

### 5. نموذج إنشاء المستخدم في لوحة الناظر
- إضافة حقل اختياري "رقم الهوية الوطنية" عند إنشاء مستخدم جديد
- ربط رقم الهوية بسجل المستفيد في جدول `beneficiaries`

### 6. Edge Function (`admin-manage-users`)
- إضافة action جديد: `toggle_registration` لتفعيل/إيقاف التسجيل
- إضافة action جديد: `lookup_by_national_id` للبحث عن مستخدم برقم الهوية
- تعديل `create_user` ليقبل `national_id` ويربطه بالمستفيد

## الملفات المتأثرة
1. Migration جديد: جدول `app_settings` + عمود `national_id`
2. `src/pages/Auth.tsx`: تعديل واجهة تسجيل الدخول
3. `src/pages/dashboard/UserManagementPage.tsx`: إضافة إعدادات التسجيل + حقل الهوية
4. `supabase/functions/admin-manage-users/index.ts`: إضافة actions جديدة
