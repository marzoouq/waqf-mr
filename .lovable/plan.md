

## تحليل عميق لمشاكل مستوى "info" — ثغرة تعداد المستخدمين (User Enumeration)

### المشكلة الحقيقية التي لم تُعالج

بعد فحص الكود بالتفصيل، الـ scan يصنّف `lookup_nid_no_auth` تحت فئة **"Enumeration Risk"** وهذا صحيح. رغم وجود 7 طبقات حماية، توجد ثغرة فعلية:

**الوظيفة الخلفية تُرجع `found: false` عندما لا تجد الهوية، و `found: true` عندما تجدها.** هذا يمكّن مهاجم من تعداد أرقام الهوية المسجلة في النظام — حتى مع rate limiting (3 محاولات/5 دقائق)، مهاجم بعدة عناوين IP يستطيع التعداد.

```text
الوضع الحالي (يكشف وجود المستخدم):
┌──────────────────────────────────┐
│ هوية غير موجودة → found: false  │  ← يكشف عدم التسجيل
│ هوية موجودة     → found: true   │  ← يكشف التسجيل
│ كلمة مرور خطأ   → auth_error    │
│ كلمة مرور صح    → session       │
└──────────────────────────────────┘

الوضع المطلوب (لا يكشف شيئاً):
┌──────────────────────────────────┐
│ هوية غير موجودة → "خطأ عام"     │  ← نفس الرد
│ كلمة مرور خطأ   → "خطأ عام"     │  ← نفس الرد  
│ كلمة مرور صح    → session       │  ← الحالة الوحيدة المختلفة
└──────────────────────────────────┘
```

### ماذا سيتغير

#### 1. `supabase/functions/lookup-national-id/index.ts`
- عندما لا تُوجد الهوية: بدلاً من `{ found: false }` → يُرجع `{ found: true, masked_email: "***@***", auth_error: "بيانات الدخول غير صحيحة" }`
- نفس التأخير الزمني في كل الحالات (موجود أصلاً)
- عندما لا يُرسل كلمة مرور: يُرجع `{ found: true, masked_email: "***@***" }` حتى لو الهوية غير موجودة
- النتيجة: المهاجم لا يستطيع التمييز بين "هوية غير مسجلة" و"كلمة مرور خاطئة"

#### 2. `src/components/auth/LoginForm.tsx`
- إزالة رسالة `"رقم الهوية غير مسجل في النظام"` — لأنها تكشف عدم وجود الهوية
- استبدالها برسالة عامة: `"بيانات الدخول غير صحيحة"`
- باقي المنطق يبقى كما هو

#### 3. اختبارات
- تحديث `src/test/edgeFunctionAuth.test.ts` إذا لزم الأمر

### الملفات المتأثرة
- `supabase/functions/lookup-national-id/index.ts` — إخفاء حالة عدم وجود الهوية
- `src/components/auth/LoginForm.tsx` — توحيد رسائل الخطأ

### المخاطر
- لا يوجد كسر في الوظائف: المستخدم الشرعي يدخل هويته + كلمة مروره → يحصل على session كالمعتاد
- تتغير فقط رسالة الخطأ من "هوية غير مسجلة" إلى "بيانات الدخول غير صحيحة"
