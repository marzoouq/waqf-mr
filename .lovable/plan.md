

# إصلاح سياسات أمان قاعدة البيانات (RLS)

## المشكلة
ثلاثة جداول تستخدم سياسة `USING (true)` التي تسمح لأي مستخدم مسجّل (حتى لو لم يكن له دور) بالاطلاع على جميع البيانات. هذا يعني أن أي شخص ينشئ حساباً يستطيع رؤية كل الإيرادات والعقود.

## التوازن المطلوب
حسب سياسة المشروع: المستفيدون يحق لهم الاطلاع على البيانات المالية (شفافية)، لكن غير المعنيين لا يحق لهم ذلك.

## الحل: تقييد الوصول للأدوار المعتمدة فقط

### التغيير الوحيد: migration SQL واحد

```text
الجدول          | السياسة الحالية              | السياسة الجديدة
----------------|-----------------------------|-----------------
income          | أي مسجّل يرى الكل          | admin + waqif + beneficiary فقط
contracts       | أي مسجّل يرى الكل          | admin + waqif + beneficiary فقط
beneficiaries   | (لا تغيير - السياسة صحيحة)  | تبقى كما هي
```

### تفاصيل SQL:

**1. جدول income:**
- حذف سياسة `Authenticated users can view income`
- إنشاء سياسة جديدة تسمح بالقراءة فقط لمن لديه دور (admin أو waqif أو beneficiary)

**2. جدول contracts:**
- حذف سياسة `Authenticated users can view contracts`
- إنشاء سياسة جديدة تسمح بالقراءة فقط لمن لديه دور (admin أو waqif أو beneficiary)

**3. جدول beneficiaries:**
- السياسة الحالية صحيحة بالفعل (admin يرى الكل + المستفيد يرى بياناته فقط)
- لا تحتاج تعديل

### لماذا نضيف beneficiary للوصول؟
لأن صفحات المستفيد (الإفصاح السنوي، التقارير المالية، حصتي) تحتاج قراءة بيانات الدخل والعقود لعرض الأرقام المالية. إذا منعنا المستفيدين سيتعطل عمل واجهاتهم.

### الملفات المتأثرة
لا توجد تغييرات على الكود - فقط migration لقاعدة البيانات.
