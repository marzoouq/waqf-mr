

# إصلاح شامل للفاتورة الضريبية — مطابقة ZATCA وفحص جنائي

## نتائج الفحص الجنائي

بعد فحص الفاتورة المُولَّدة (`فاتورة-10821806867-R1-INV-07.pdf`) والمرجع (وافق/Wafiq):

| العنصر | الحالة | السبب الجذري |
|--------|--------|-------------|
| **QR Code** | ❌ غائب تماماً | `generateQrDataUrl` تعتمد على `qrcode` library — قد تفشل صامتاً. الـ fallback يرسم نصاً صغيراً لا يُرى |
| **بيانات البائع** (ضريبي، CR، عنوان) | ❌ غائبة | الحقول شرطية `if (waqfInfo?.vatNumber)` — إذا لم تُملأ في الإعدادات لا يظهر شيء ولا تنبيه على PDF |
| **الشعار** | ❌ غائب | نفس السبب — `waqfInfo.logoUrl` فارغ |
| **QR موقعه خاطئ** | ⚠️ | في القالب الضريبي يُرسم عند `(margin, y+10)` لكن إذا فشل `addImage` لا يظهر شيء مرئي |
| **عنوان ثنائي اللغة** | ❌ | ZATCA تتطلب "فاتورة ضريبية / Tax Invoice" |
| **رقم الفاتورة في الترويسة** | ⚠️ | المرجع يضعه بارزاً بجانب العنوان |

## التعديلات المطلوبة

### 1. إصلاح QR Code — `paymentInvoice.ts`
- المشكلة: `qrcode` library قد تفشل في بيئة المتصفح عند بناء Data URL
- الحل: إضافة **retry logic** + fallback برسم مربع QR placeholder واضح مع نص "يرجى إضافة بيانات الوقف"
- إضافة `console.error` مفصّل لتتبع الخطأ

### 2. إعادة هيكلة ترويسة `renderTaxProfessional`
التخطيط الجديد مطابق لمعيار ZATCA:

```text
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  فاتورة ضريبية / Tax Invoice    [اسم الوقف]    [شعار]   │
│  Invoice No: INV-xxx             الرقم الضريبي: xxx     │
│  Date: 2026-03-14                السجل التجاري: xxx     │
│                                  العنوان: xxx            │
│  [QR Code 35x35]                                        │
│                                                          │
├──────────────────────────────────────────────────────────┤
│ بيانات العميل           │    بيانات الفاتورة             │
│ الاسم: xxx              │    رقم العقد: xxx              │
│ الرقم الضريبي: xxx      │    العقار: xxx                 │
│                         │    الدفعة: x من x              │
├──────────────────────────────────────────────────────────┤
│ جدول البنود (8 أعمدة)                                    │
├──────────────────────────────────────────────────────────┤
│ ملخص: قبل الضريبة | الضريبة | الإجمالي                   │
├──────────────────────────────────────────────────────────┤
│ بيانات الدفع: بنك | حساب | IBAN                          │
└──────────────────────────────────────────────────────────┘
```

- **عنوان ثنائي اللغة**: "فاتورة ضريبية" يمين + "Tax Invoice" يسار
- **رقم الفاتورة + التاريخ** بارزان في الترويسة (ليسا في قسم منفصل)
- **QR Code** تحت العنوان مباشرة بحجم 35x35mm
- **بيانات البائع** على اليمين مع placeholder واضح إذا كانت فارغة: "(غير مُعرَّف)" بلون رمادي
- **الشعار** أقصى اليمين (كما هو)

### 3. إضافة placeholder للبيانات الناقصة في PDF
بدلاً من إخفاء السطر إذا كان الحقل فارغاً:
- `الرقم الضريبي: (غير مُعرَّف)` بلون أحمر خافت
- هذا يجعل المستخدم يعرف أن الحقل ناقص حتى في PDF نفسه

### 4. تحسين `renderQrCode` بـ retry + visible fallback
- محاولة أولى: `generateQrDataUrl(tlvBase64)`
- محاولة ثانية (إذا فشلت): `QRCode.toDataURL` مباشرة مع options مختلفة
- fallback مرئي: رسم مربع بإطار منقط مع نص "QR" واضح بحجم كبير
