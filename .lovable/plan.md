### 1. الأمن (Security) — تقييم: 9.5/10

| البند | الحالة | التفاصيل |
|---|---|---|
| العروض الآمنة (Views) | ✅ سليم | `beneficiaries_safe` و `contracts_safe` مضبوطتان بـ `security_invoker=true` + `security_barrier=true` — التنبيهات الأمنية التي تظهر هي **إنذارات كاذبة** لأن الماسح لا يكتشف خصائص العرض |
| صلاحيات `anon` | ✅ سليم | `anon` ممنوع من الوصول للعروض الآمنة (تم التحقق: `false`) |
| صلاحيات `authenticated` | ✅ سليم | مقيد بـ `SELECT` فقط مع تطبيق RLS من الجدول الأصلي |
| Edge Functions | ✅ سليم | جميع الدوال تستخدم المصادقة اليدوية الصحيحة (`getUser()` أو service role للعمليات الإدارية) |
| `guard-signup` | ✅ سليم | يتضمن rate limiting + input validation + rollback عند فشل تعيين الدور |
| تسريب أخطاء | ✅ سليم | رسائل الخطأ موحدة ولا تكشف معلومات داخلية (`console.error` بدون تفاصيل) |

**ملاحظة أمنية**: تنبيه `Extension in Public` (pgcrypto) معروف ومقبول — الإضافات معزولة في schema `extensions` والدوال تضمّن المسار الصحيح.
