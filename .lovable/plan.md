
# إصلاح بطاقات الإشغال لتتغير حسب السنة المالية

## المشكلة

البطاقات الأربع في أعلى صفحة العقارات للمستفيد تعرض أرقام ثابتة لا تتغير مع تبديل السنة المالية:

```
إجمالي العقارات: 5 (ثابت)
إجمالي الوحدات: 23 (ثابت)
مؤجرة: 15 (ثابت - يعتمد على unit.status)
شاغرة: 8 (ثابت - يعتمد على unit.status)
```

**السبب:** السطر 52 في `PropertiesViewPage.tsx`:
```
const occupiedUnits = units?.filter(u => u.status === 'مؤجرة').length ?? 0;
```
هذا يقرأ حالة الوحدة الثابتة ولا يستخدم العقود المفلترة بالسنة.

## الحل

تغيير منطق حساب "مؤجرة" و"شاغرة" ليعتمد على **العقود المفلترة بالسنة المالية** بدلا من حالة الوحدة الثابتة.

### تعديل `src/pages/beneficiary/PropertiesViewPage.tsx`:

1. **حساب الوحدات المؤجرة من العقود:**
   - جمع كل `unit_id` و `property_id` من العقود المفلترة بالسنة
   - الوحدة تعتبر "مؤجرة" في سنة معينة إذا كان لها عقد مرتبط بتلك السنة
   - العقارات بدون وحدات تعتبر "مؤجرة بالكامل" إذا كان لها عقد مباشر

2. **تحديث البطاقات العلوية:**
   - "إجمالي العقارات" و "إجمالي الوحدات": تبقى ثابتة (أصول لا تتغير)
   - "مؤجرة": عدد الوحدات التي لها عقود في السنة المختارة
   - "شاغرة": إجمالي الوحدات - المؤجرة (حسب السنة)

3. **المنطق:**
   ```
   // جمع الوحدات التي لها عقود في السنة المختارة
   const rentedUnitIds = new Set(
     contracts.filter(c => c.unit_id).map(c => c.unit_id)
   );
   // العقارات المؤجرة بالكامل (بدون وحدات)
   const wholePropertyIds = new Set(
     contracts.filter(c => c.property_id && !c.unit_id).map(c => c.property_id)
   );
   // الحساب: وحدة مؤجرة = لها عقد في السنة، أو عقارها مؤجر بالكامل
   const occupiedUnits = units.filter(u =>
     rentedUnitIds.has(u.id) || wholePropertyIds.has(u.property_id)
   ).length;
   ```

هذا يضمن أن تبديل السنة المالية يغير أرقام الإشغال فورا لتعكس واقع تلك السنة تحديدا.
