

# اصلاح منهجي شامل لحسابات الايجارات - معايير محاسبية مؤسسية

## المبدأ المحاسبي الصحيح

في المحاسبة المؤسسية والمعايير الدولية (IFRS/GAAP)، لا يتم "حذف" أو "شطب" أي عقد من السجلات المالية. كل عقد - نشط أو منتهي - يمثل دخلا فعليا تم تحصيله ويجب ان يظهر بشفافية كاملة.

## المشاكل الحالية

### 1. بطاقة العقار تحسب العقود النشطة فقط
السطر 222-223 يفلتر `status === 'active'` فقط، مما يعني ان دخل العقود المنتهية لا يظهر نهائيا في ملخص العقار.

### 2. الايجار الشهري يقسم على 12 دائما
السطر 814 يحسب `rent_amount / 12` بغض النظر عن نوع الدفع (شهري/سنوي/متعدد). هذا خاطئ محاسبيا.

### 3. `getTenant` يعرض عقد واحد فقط لكل وحدة
اذا كان للوحدة عقد منتهي ثم عقد نشط، يعرض النشط فقط. العقد المنتهي يختفي.

## الحل المنهجي المؤسسي

### ملف واحد: `src/pages/dashboard/PropertiesPage.tsx`

### اولا: بطاقة العقار - عرض مؤشرين ماليين منفصلين (سطور 222-228)

**قبل:**
```text
الشهري: X ريال
السنوي: Y ريال
```

**بعد:** عرض سطرين ماليين واضحين:

| المؤشر | المعنى | المصدر |
|--------|--------|--------|
| الايرادات التعاقدية | اجمالي كل العقود (نشطة + منتهية) | جميع العقود المرتبطة بالعقار |
| الدخل النشط الحالي | ما يُحصّل فعليا الان | العقود النشطة فقط |
| المصروفات | اجمالي المصاريف | كما هو |
| الصافي | الايرادات التعاقدية - المصروفات | كما هو لكن يشمل الكل |

هذا يضمن ان كل ريال من الدخل محسوب ومعروض بشفافية.

### ثانيا: اصلاح حساب الايجار الشهري (سطر 814)

**قبل:** `tenant.rent_amount / 12` (خاطئ)

**بعد:** استخدام `payment_amount` من العقد مباشرة:
- اذا كان `payment_type === 'monthly'`: الشهري = `payment_amount` او `rent_amount / 12`
- اذا كان `payment_type === 'annual'`: الشهري = `rent_amount / 12`
- اذا كان `payment_type === 'multiple'`: الشهري = `payment_amount` (قيمة الدفعة الواحدة)

وتحديث `getTenant` لاضافة `payment_type` و `payment_amount` و `payment_count` من بيانات العقد.

### ثالثا: صف اجمالي اسفل جدول الوحدات (بعد سطر 848)

اضافة صف ملخص بخلفية مميزة يعرض:
- **اجمالي الايجارات السنوية** (جميع العقود)
- **اجمالي الايجارات الشهرية** (جميع العقود)
- وسم يوضح "شامل النشط والمنتهي"

لا يتم شطب او اخفاء اي مبلغ. كل عقد يظهر بقيمته الكاملة مع وسم حالته فقط.

### رابعا: بطاقة العقار - تحديث حساب الشهري (سطر 224)

**قبل:** `annualRent / 12`

**بعد:** جمع الشهري الفعلي لكل عقد حسب نوع الدفع بدلا من القسمة الثابتة:
```text
monthlyRent = contracts.reduce((sum, c) => {
  if (c.payment_type === 'monthly') return sum + (c.payment_amount || c.rent_amount / 12)
  if (c.payment_type === 'multiple') return sum + (c.payment_amount || c.rent_amount / c.payment_count)
  return sum + c.rent_amount / 12  // annual
}, 0)
```
