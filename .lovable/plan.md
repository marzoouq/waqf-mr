

# إصلاح الفاتورة الضريبية — مطابقة المرجع وامتثال ZATCA

## المشاكل المكتشفة

بعد فحص الفاتورة المُولَّدة مقابل المرجع (فاتورة دفترة):

| المشكلة | السبب |
|---------|-------|
| **QR Code غير ظاهر** | الدالة `renderQrCode` تعمل لكن موقعها يتداخل مع عنوان "فاتورة ضريبية" — والخطأ يُبتلع بصمت |
| **بيانات الوقف ناقصة** (ضريبي، سجل تجاري، عنوان) | الحقول شرطية — تظهر فقط إذا أُدخلت في الإعدادات، لكن لا يوجد تنبيه للمستخدم |
| **الشعار يتداخل مع النص** | موقع الشعار `(pageW - margin - 55, 16)` يتعارض مع بيانات البائع |
| **تخطيط لا يطابق المرجع** | المرجع يضع: QR أسفل يسار العنوان، بيانات البائع وسط-يمين، الشعار أقصى اليمين |

## التعديلات المطلوبة

### 1. إصلاح `renderTaxProfessional` — `paymentInvoice.ts`
إعادة ترتيب الترويسة لتطابق المرجع:

```text
┌──────────────────────────────────────────────────┐
│ فاتورة ضريبية    اسم الوقف              [شعار]  │
│                  الرقم الضريبي: xxx              │
│  [QR 35x35]      السجل التجاري: xxx             │
│                  العنوان: xxx                    │
├──────────────────────────────────────────────────┤
```

- نقل الشعار لأقصى اليمين (بجانب الاسم)
- نقل QR تحت عنوان "فاتورة ضريبية" مباشرة
- إضافة `try/catch` مع `console.warn` حول QR لكشف الأخطاء
- إضافة خطوط فاصلة ذهبية/خضراء للإطار

### 2. إضافة تنبيه بيانات ناقصة — `PaymentInvoicesTab.tsx`
- إذا كان `waqfInfo.vatNumber` أو `waqfInfo.commercialReg` أو `waqfInfo.address` فارغاً، يظهر تنبيه (Alert) أصفر:
  "لضمان امتثال الفاتورة الضريبية، يرجى إكمال بيانات المنشأة في الإعدادات (الرقم الضريبي، السجل التجاري، العنوان)"
  مع رابط مباشر لصفحة الإعدادات

### 3. تحسين `renderQrCode` — إضافة logging وfallback
- إذا فشل `generateQrDataUrl`، عرض نص TLV مختصر كـ fallback (موجود فعلاً لكن بحاجة لتفعيل أفضل)
- إضافة `console.warn` لتتبع الأخطاء

### 4. تحديث الاختبارات
