

# خطة: لوحة تحكم شاملة لصلاحيات التطبيق

## تحليل الوضع الحالي

الناظر لديه حالياً **4 تبويبات مبعثرة** في الإعدادات تتحكم بالصلاحيات:

| التبويب | ما يفعله | المشكلة |
|---------|----------|---------|
| **صلاحيات الأدوار** | تفعيل/تعطيل أقسام لكل دور | يتحكم فقط بإظهار/إخفاء القائمة |
| **الأقسام** | إظهار/إخفاء أقسام الناظر والمحاسب | مكرر جزئياً مع صلاحيات الأدوار |
| **واجهة المستفيد** | إظهار/إخفاء أقسام المستفيد | مفصول عن باقي الصلاحيات |
| **القائمة** | تغيير أسماء عناصر القائمة | شكلي فقط |

**الثغرات المكتشفة:**
1. لا توجد صلاحيات على مستوى **العمليات** (قراءة/إضافة/تعديل/حذف) — فقط إظهار/إخفاء القسم
2. لا يوجد سجل لتتبع **من غيّر أي صلاحية ومتى**
3. لا توجد حماية على مستوى المسارات (URL) — إخفاء القائمة فقط، يمكن الوصول المباشر عبر URL
4. تداخل بين "الأقسام" و"صلاحيات الأدوار" يسبب ارتباكاً

---

## ما سيُنفذ

### 1. دمج التبويبات الأربعة في لوحة صلاحيات موحدة
- تبويب واحد بعنوان **"إدارة الصلاحيات"** يحل محل التبويبات المتفرقة
- مصفوفة تفاعلية: الأدوار (أعمدة) × الأقسام (صفوف) مع خانات اختيار
- زر "تحديد الكل / إلغاء الكل" لكل دور
- يدمج: sections_visibility + beneficiary_sections + role_permissions في واجهة واحدة

### 2. إضافة حارس مسارات (Route Guard)
- مكوّن `RequirePermission` يمنع الوصول المباشر عبر URL إذا كان القسم مُعطّلاً
- يعيد التوجيه للصفحة الرئيسية مع رسالة toast
- يُستثنى الناظر (admin) دائماً

### 3. ملخص مرئي للصلاحيات
- بطاقة ملخص في أعلى اللوحة تُظهر: عدد الأقسام المفعّلة لكل دور
- مؤشر بصري (شريط تقدم) لنطاق الصلاحيات

### 4. تسجيل تغييرات الصلاحيات في سجل المراجعة
- عند حفظ أي تغيير في الصلاحيات، يُسجّل في `access_log` مع تفاصيل التغيير

## التفاصيل التقنية

### الملفات الجديدة
| الملف | الغرض |
|-------|-------|
| `src/components/settings/PermissionsControlPanel.tsx` | اللوحة الموحدة الجديدة |
| `src/components/guards/RequirePermission.tsx` | حارس المسارات |

### الملفات المعدّلة
| الملف | التغيير |
|-------|---------|
| `src/pages/dashboard/SettingsPage.tsx` | استبدال 4 تبويبات بتبويب واحد "إدارة الصلاحيات" |
| `src/hooks/page/useNavLinks.ts` | إضافة تصدير دالة `usePermissionCheck` لحارس المسارات |
| `src/constants/rolePermissions.ts` | لا تغيير — يبقى كمصدر للقيم الافتراضية |
| `src/constants/sections.ts` | لا تغيير — يبقى كمصدر لتعريفات الأقسام |
| ملفات المسارات (Router) | تغليف المسارات المحمية بـ `RequirePermission` |

### منطق العمل
- القراءة/الكتابة تستخدم نفس `useAppSettings` و`app_settings` الحالية
- المفاتيح المحفوظة تبقى كما هي (`role_permissions`, `sections_visibility`, `beneficiary_sections`) للتوافقية
- اللوحة الجديدة تقرأ وتكتب في نفس المفاتيح الثلاثة لكن من واجهة واحدة
- التبويبات القديمة الثلاثة (`SectionsTab`, `BeneficiaryTab`, `RolePermissionsTab`) تُحذف من صفحة الإعدادات
- `MenuCustomizationTab` يبقى منفصلاً (تغيير الأسماء ≠ صلاحيات)
