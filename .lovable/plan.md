

# اصلاح مشكلة تسجيل الدخول برقم الهوية

## نتائج الفحص العميق المؤكدة

| الفحص | النتيجة |
|-------|---------|
| استدعاء `lookup-national-id` مباشرة في الإنتاج | **نجح** — أرجع جلسة صحيحة |
| كلمة المرور `Aa05459898133@` | **صحيحة** في الإنتاج |
| المستفيد في جدول `beneficiaries` | موجود ومرتبط بـ `user_id` |
| دور المستخدم في `user_roles` | `beneficiary` — صحيح |
| دالة `lookup_by_national_id` في الإنتاج | موجودة |
| سجلات `admin-manage-users` | "Password verify login SUCCESS" |
| سجلات Auth | آخر أخطاء `invalid_credentials` قبل إعادة التعيين فقط |

## السبب الجذري

**حد المحاولات (Rate Limit) منخفض جدا + رسائل الخطأ مبهمة**

- الحد الحالي: **3 محاولات كل 5 دقائق**
- بعد 3 محاولات (فاشلة أو ناجحة): الوظيفة ترجع خطأ 429
- معالجة الخطأ 429 في `LoginForm` غير دقيقة — تظهر رسالة عامة بدل رسالة واضحة عن تجاوز الحد
- المستخدم يرى "حدث خطأ في الاتصال" أو رسالة مبهمة بدل "تجاوزت حد المحاولات، انتظر X ثانية"
- بالإضافة: حقل كلمة المرور بدون زر إظهار يزيد احتمال الخطأ في الرموز

## خطة الإصلاح

### 1. رفع حد المحاولات
**الملف**: `supabase/functions/lookup-national-id/index.ts` (سطر 4-5)

```
RATE_LIMIT = 3 → 5
RATE_WINDOW_SECONDS = 300 → 180
```

### 2. إصلاح معالجة خطأ Rate Limit في واجهة تسجيل الدخول
**الملف**: `src/components/auth/LoginForm.tsx` (سطر 74-91)

المشكلة: عند رجوع خطأ 429 من `supabase.functions.invoke`، المعالجة الحالية تفحص `lookupError.message` لكن Supabase SDK يضع الخطأ بشكل مختلف عند non-2xx. الإصلاح:
- فحص `data?.remaining === 0` أو `data?.error` التي تحتوي "تم تجاوز حد المحاولات"
- إذا كان `lookupError` من نوع `FunctionsHttpError`: محاولة قراءة body كـ JSON لاستخراج الرسالة الصحيحة
- عرض رسالة واضحة: "تم تجاوز حد المحاولات. يرجى الانتظار X ثانية"

### 3. إضافة زر إظهار/إخفاء كلمة المرور
**الملف**: `src/components/auth/LoginForm.tsx` (سطر 253-262)

- إضافة زر Eye/EyeOff بجانب حقل كلمة المرور
- تبديل `type` بين `password` و `text`
- يمنع أخطاء إدخال الرموز مثل `@` و `@@`

### 4. نشر الوظائف بعد التعديل
- إعادة نشر `lookup-national-id` لتفعيل الحد الجديد

## النتيجة المتوقعة

```text
المستخدم يدخل رقم الهوية + كلمة المرور
→ يمكنه إظهار كلمة المرور للتأكد
→ 5 محاولات كل 3 دقائق (بدل 3 كل 5 دقائق)
→ إذا تجاوز الحد: رسالة واضحة بعدد الثواني المتبقية
→ إذا كانت كلمة المرور صحيحة: يدخل فورا
```
