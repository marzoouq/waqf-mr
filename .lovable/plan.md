

# خطة إضافة تقرير PDF للفحص الجنائي

## الوصف
إنشاء دالة توليد تقرير PDF رسمي يلخص نتائج الفحص الجنائي (Forensic Audit) للنظام، مع توقيع رقمي (اسم الناظر + التاريخ) وتصميم متوافق مع باقي تقارير النظام.

## محتوى التقرير
التقرير يتضمن الأقسام التالية:
1. **عنوان**: "تقرير الفحص الجنائي - نظام إدارة الوقف"
2. **ملخص تنفيذي**: التقييم العام (9.8/10)، عدد الملفات المفحوصة، عدد المشاكل المكتشفة والمصححة
3. **جدول نتائج الفحص**: المجال، الحالة (سليم/مُصحح/ملاحظة)، التفاصيل
4. **قسم الأمان**: نتائج فحص RLS وسياسات الأدوار
5. **التوقيع الرقمي**: اسم الناظر، التاريخ الهجري/الميلادي، ختم "مُعتمد"

## التفاصيل التقنية

### ملف جديد: `src/utils/pdf/forensicAudit.ts`
- دالة `generateForensicAuditPDF(data, waqfInfo)` تتبع نفس نمط التقارير الأخرى
- تستخدم `loadArabicFont`, `addHeader`, `addFooter`, `addHeaderToAllPages` من `core.ts`
- تستخدم `autoTable` لعرض الجداول مع ألوان مخصصة (أخضر للسليم، ذهبي للملاحظات)

### واجهة البيانات (Interface):
```text
ForensicAuditData:
  - auditDate: string (تاريخ الفحص)
  - auditorName: string (اسم المدقق/الناظر)
  - overallScore: number (التقييم من 10)
  - totalFiles: number
  - issuesFound: number
  - issuesFixed: number
  - categories: مصفوفة من {category, status, details, score}
  - securityFindings: مصفوفة من {finding, severity, status, notes}
```

### تعديل: `src/utils/pdf/index.ts`
- إضافة تصدير `generateForensicAuditPDF` من الملف الجديد

### تعديل: `src/pages/dashboard/ReportsPage.tsx`
- إضافة زر "تقرير الفحص الجنائي PDF" في شريط الأزرار العلوي
- الزر يولد التقرير ببيانات ثابتة تعكس نتائج الفحص الفعلية

### قسم التوقيع الرقمي في PDF:
- مربع رسمي أسفل التقرير يحتوي على:
  - "أُعد بواسطة: نظام إدارة الوقف - الفحص الجنائي الآلي"
  - "اعتمده: [اسم الناظر من إعدادات الوقف]"
  - "التاريخ: [التاريخ الميلادي]"
  - ختم دائري بنص "مُعتمد" بلون أخضر داكن
