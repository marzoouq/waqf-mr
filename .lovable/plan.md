
# إنشاء ملفات PDF مرفقة للفواتير الـ 6 الجديدة

## الفكرة

إنشاء Edge Function تقوم بتوليد ملفات PDF احترافية لكل فاتورة من الفواتير الـ 6، ثم رفعها إلى مستوعب التخزين `invoices` وتحديث سجلات الفواتير بمسار الملف واسمه.

## الفواتير المستهدفة

| رقم الفاتورة | الوصف | المبلغ (ر.س) |
|--------------|-------|-------------|
| INV-2025-008 | أعمال جبس | 30,127.25 |
| INV-2025-009 | أعمال سباكة | 24,259.00 |
| INV-2025-010 | أعمال كهرباء | 9,205.76 |
| INV-2025-011 | عامل نظافة شهري | 3,756.37 |
| INV-2025-012 | تدقيق محاسبي 2024 | 2,501.15 |
| INV-2025-013 | تدقيق محاسبي 2025 | 2,240.00 |

## الحل التقني

### 1. إنشاء Edge Function: `generate-invoice-pdf`

دالة خلفية تقوم بـ:
- استقبال قائمة بمعرّفات الفواتير (أو توليدها جميعاً دفعة واحدة)
- جلب بيانات كل فاتورة من قاعدة البيانات
- توليد ملف PDF بسيط لكل فاتورة يحتوي على:
  - اسم الوقف: "وقف مرزوق بن علي الثبيتي"
  - رقم الصك: ٤١١٢٠٩٧٠٧
  - المحكمة: محكمة الأحوال الشخصية بالطائف
  - اسم الناظر: عبدالله بن مرزوق بن علي الثبيتي
  - رقم الفاتورة والنوع والمبلغ والتاريخ والوصف
- رفع ملف PDF إلى مستوعب `invoices`
- تحديث سجل الفاتورة بحقلي `file_path` و `file_name`

### 2. استدعاء الدالة

بعد نشر الدالة، سيتم استدعاؤها مباشرة لتوليد الملفات الـ 6.

### 3. إضافة زر في واجهة الفواتير (اختياري)

إضافة زر "توليد PDF" في صفحة الفواتير للفواتير التي لا تحتوي على مرفق، لتمكين الناظر من توليد فواتير مستقبلاً.

## الملفات المتأثرة

| الملف | التغيير |
|-------|---------|
| `supabase/functions/generate-invoice-pdf/index.ts` | إنشاء جديد - Edge Function لتوليد PDF |
| `src/pages/dashboard/InvoicesPage.tsx` | إضافة زر "توليد PDF" للفواتير بدون مرفق |
| `src/hooks/useInvoices.ts` | إضافة hook لاستدعاء دالة التوليد |
| قاعدة البيانات | تحديث `file_path` و `file_name` للفواتير الـ 6 |

## تفاصيل تصميم ملف PDF

كل ملف PDF يحتوي على:
- ترويسة: اسم الوقف + رقم الصك + المحكمة
- عنوان: "فاتورة" + رقم الفاتورة
- جدول بسيط: النوع، المبلغ، التاريخ، الوصف، الحالة
- تذييل: اسم الناظر + التاريخ

سيتم استخدام مكتبة مناسبة لبيئة Deno لتوليد PDF داخل Edge Function (مثل `jsPDF` المتوفرة عبر esm.sh).
