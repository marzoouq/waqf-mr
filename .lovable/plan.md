

# إضافة عقود متعددة الوحدات لمستأجر واحد

## المشكلة الحالية

نموذج العقد الحالي يسمح فقط باختيار **وحدة واحدة** أو **العقار كامل**. إذا استأجر شخص 4 وحدات، يجب تكرار النموذج 4 مرات يدوياً مع إعادة إدخال نفس البيانات (اسم المستأجر، التواريخ، نوع الدفع...).

## الحل المقترح

تطوير نموذج العقد ليدعم **3 أوضاع تأجير** بمرونة:

```text
+------------------------------------------+
|          نوع التأجير                      |
|  [العقار كامل] [وحدة واحدة] [وحدات متعددة]|
+------------------------------------------+
|                                          |
|  عند اختيار "وحدات متعددة":              |
|  +------------------------------------+  |
|  | [ ] معرض تجاري    (الأرضي)        |  |
|  | [x] شقة 1         (الأول)         |  |
|  | [x] شقة 2         (الأول)         |  |
|  | [x] شقة 3         (الثاني)        |  |
|  | [x] شقة 4         (الثاني)        |  |
|  +------------------------------------+  |
|  تم اختيار: 4 وحدات                     |
|                                          |
|  نوع الإيجار:                            |
|  (*) إيجار إجمالي (مبلغ واحد لكل الوحدات)|
|  ( ) إيجار لكل وحدة (تحديد مبلغ لكل وحدة)|
|                                          |
|  [عند اختيار إيجار إجمالي]               |
|  الإيجار الإجمالي: [40,000] ر.س          |
|  = 10,000 ر.س لكل وحدة                  |
|                                          |
|  [عند اختيار إيجار لكل وحدة]             |
|  شقة 1: [12,000] ر.س                    |
|  شقة 2: [10,000] ر.س                    |
|  شقة 3: [9,000] ر.س                     |
|  شقة 4: [9,000] ر.س                     |
|                                          |
|  رقم العقد الأساسي: C-2026-005          |
|  -> سيتم إنشاء: C-2026-005-A,B,C,D     |
|                                          |
|  [إنشاء 4 عقود]              [إلغاء]    |
+------------------------------------------+
```

## آلية العمل

1. **اختيار العقار** -- كالمعتاد
2. **اختيار نوع التأجير** -- 3 خيارات بدل 2:
   - العقار كامل (unit_id = null)
   - وحدة واحدة (Select كالحالي)
   - وحدات متعددة (Checkboxes)
3. **عند اختيار "وحدات متعددة"**:
   - تظهر قائمة الوحدات كـ Checkboxes
   - الوحدات المؤجرة بالفعل (لديها عقد نشط) تظهر معطلة مع تلميح
   - اختيار طريقة تسعير الإيجار (إجمالي أو لكل وحدة)
4. **عند الإرسال**:
   - يتم إنشاء عقد منفصل لكل وحدة محددة
   - ترقيم تلقائي: إذا كان الرقم الأساسي `C-2026-005`، تصبح العقود `C-2026-005-A`, `C-2026-005-B`, إلخ
   - نفس بيانات المستأجر والتواريخ ونوع الدفع لجميع العقود
   - رسالة نجاح واحدة: "تم إنشاء 4 عقود للمستأجر [اسم]"

## التفاصيل التقنية

### 1. تحديث ContractFormData

إضافة حقول جديدة لدعم الوضع المتعدد:

- `rental_mode`: `'full' | 'single' | 'multi'` -- نوع التأجير
- `selected_unit_ids`: `string[]` -- الوحدات المختارة (وضع متعدد)
- `rent_per_unit`: `Record<string, string>` -- إيجار كل وحدة (اختياري)
- `pricing_mode`: `'total' | 'per_unit'` -- طريقة التسعير

### 2. تحديث ContractFormDialog.tsx

- إضافة مكون اختيار نوع التأجير (ToggleGroup أو RadioGroup)
- إضافة قائمة Checkboxes للوحدات مع عرض حالة كل وحدة
- إضافة خيار التسعير (إجمالي / لكل وحدة)
- تحديث منطق handleSubmit لإنشاء عقود متعددة عبر loop

### 3. تحديث ContractsPage.tsx - handleFormSubmit

- التعامل مع الوضع المتعدد: إنشاء عقد لكل وحدة محددة
- ترقيم العقود تلقائياً بلاحقة (A, B, C...)
- استخدام `Promise.all` أو loop متتالي لإنشاء العقود
- عرض رسالة نجاح موحدة

### 4. تصفية الوحدات المتاحة

- جلب العقود النشطة للعقار المختار
- تعطيل الوحدات التي لديها عقد نشط بالفعل
- عرض اسم المستأجر الحالي كتلميح على الوحدة المعطلة

### الملفات المتأثرة:
- `src/components/contracts/ContractFormDialog.tsx` -- التعديل الرئيسي
- `src/pages/dashboard/ContractsPage.tsx` -- تعديل handleFormSubmit

### لا تغييرات مطلوبة في:
- قاعدة البيانات (نفس جدول contracts)
- الـ hooks (نفس useCreateContract)
- الـ triggers (sync_unit_status يعمل لكل عقد)
