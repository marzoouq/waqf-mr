## تحليل الوضع الحالي

- `package.json` → الإصدار `2.6.0`
- `changelog.json` → يحتوي بالفعل على إدخال `2.7.0` بتاريخ ١٢ مارس ٢٠٢٦ مع التغييرات الأخيرة
- GitHub Action (`auto-version.yml`) → يرفع patch تلقائياً عند الدفع لـ `main`، لكنه يرفع patch فقط (مثلاً 2.6.0 → 2.6.1) ولا يرفع minor
- تنبيه `allocate_icv_and_chain` → تم إغلاقه ✅

## المشكلة

`package.json` ما زال `2.6.0` بينما `changelog.json` يسجّل `2.7.0`. هذا يعني:

1. `VITE_APP_VERSION` = `2.6.0` → نظام PWA لا يكتشف تحديث
2. المستخدمون لا يرون toast التحديث ولا سجل التغييرات

## الخطة

### تغيير واحد فقط:

**رفع `version` في `package.json` من `2.6.0` إلى `2.7.0**`

هذا سيُفعّل:

- `VITE_APP_VERSION` = `2.7.0`
- مقارنة localStorage: `2.6.0 ≠ 2.7.0` → مسح الكاش → إعادة تحميل → عرض toast التحديث
- `changelog.json` جاهز بالفعل بإدخال `2.7.0`

### ملاحظة عن GitHub Action

الـ Action يرفع patch عند كل push لـ `main` (سيصبح `2.7.1` عند أول push بعد ذلك). هذا سلوك طبيعي ومتوقع.  
  
ضروري تحقق من بيئية الانتاج  بانها تطابق بيئية التطوير حيث هناك تحديثات لم تظهر في بئية الانتاج
