

# تقرير الفحص الجنائي — السنة المالية المقفلة 2024-2025

## ملخص تنفيذي

تم فحص كافة البيانات المالية للسنة المقفلة (2024-10-25 إلى 2025-10-24). النتيجة: **لا يوجد تداخل في المصروفات**، لكن تم اكتشاف 4 مشاكل أخرى تتطلب إصلاحاً.

---

## 1. المصروفات — لا تداخل ✅

```text
عدد المصروفات: 13 سجل
إجمالي المصروفات: 121,722.10 ر.س
تاريخ جميعها: 2025-06-01 (ضمن نطاق السنة المالية)
المبلغ المخزن بالحساب الختامي: 121,722.10 ✅ متطابق
```

جميع المصروفات مرتبطة بالسنة المالية الصحيحة، ولا يوجد أي مصروف يتبع سنة مالية خاطئة.

---

## 2. المشاكل المكتشفة

### مشكلة A: التوزيعات بدون سنة مالية (خطورة عالية)

```text
14 توزيع بمبلغ إجمالي 995,000.03 ر.س ← fiscal_year_id = NULL
الحساب الختامي يسجل distributions_amount = 995,000.03 ← متطابق
لكن التوزيعات غير مربوطة بالسنة المالية
```

**التأثير**: عند فلترة التوزيعات بالسنة المالية، لا تظهر هذه التوزيعات. صفحة "حصتي" للمستفيد وتقرير الإفصاح لن يعرضا هذه البيانات بشكل صحيح.

**الإصلاح**: ربط التوزيعات اليتيمة بالسنة المقفلة (تاريخها 2025-10-25 يتطابق مع يوم الإقفال):
```sql
UPDATE distributions 
SET fiscal_year_id = '1fe1394b-...' 
WHERE fiscal_year_id IS NULL
```

### مشكلة B: العقارات السكنية لم تُفعّل كمعفاة من الضريبة (خطورة عالية)

```text
العقار 101 (مبنى سكني)  → vat_exempt = false ❌
العقار 102 (شقق سكنية)  → vat_exempt = false ❌
```

أضفنا عمود `vat_exempt` في الجلسة السابقة لكن لم نحدّث العقارات السكنية القائمة.

**التأثير**: عند إعادة توليد الفواتير، ستظل تحتسب ضريبة 15% على السكني.

**الإصلاح**:
```sql
UPDATE properties SET vat_exempt = true 
WHERE property_type IN ('مبنى سكني', 'شقق سكنية')
```

### مشكلة C: فواتير سكنية بضريبة خاطئة (خطورة عالية)

```text
~90 فاتورة دفعة (payment_invoices) لعقارات سكنية تحمل vat_rate = 15%
المفترض: vat_rate = 0% (سكني معفى)
```

**الإصلاح**: تصفير الضريبة على فواتير العقارات السكنية التي لم تُدفع/تُوقّع:
```sql
UPDATE payment_invoices pi
SET vat_rate = 0, vat_amount = 0
FROM contracts c
JOIN properties p ON p.id = c.property_id
WHERE pi.contract_id = c.id
  AND p.property_type IN ('مبنى سكني', 'شقق سكنية')
  AND pi.zatca_status IS DISTINCT FROM 'reported'
  AND pi.icv IS NULL
```

### مشكلة D: 18 فاتورة دفعة بدون سنة مالية

```text
18 payment_invoices مع fiscal_year_id = NULL
```

تم إصلاح الدالة في الجلسة السابقة (fallback)، لكن الفواتير الحالية تحتاج تحديثاً يدوياً.

**الإصلاح**:
```sql
UPDATE payment_invoices pi
SET fiscal_year_id = c.fiscal_year_id
FROM contracts c
WHERE pi.contract_id = c.id AND pi.fiscal_year_id IS NULL
```
