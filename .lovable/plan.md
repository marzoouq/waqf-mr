
# إصلاح تواريخ تجديد العقود + تصحيح العقود المُجددة خطأً

## المشكلة
دالة التجديد تستخدم `new Date()` (تاريخ اليوم) كتاريخ بداية للعقد الجديد، بينما المطلوب أن يبدأ العقد الجديد من **تاريخ انتهاء العقد القديم** وبنفس **مدة العقد الأصلي**.

### مثال على الخطأ الحالي:
| العقد القديم | انتهاء | العقد الجديد (خطأ) | العقد الجديد (صحيح) |
|---|---|---|---|
| 10102023663 | 2025-10-26 | 2026-02-23 → 2027-02-22 | 2025-10-26 → 2026-10-26 |
| 10666673386 | 2025-11-30 | 2026-02-23 → 2027-02-22 | 2025-11-30 → 2026-11-30 |

## الحل - جزءان:

### الجزء 1: إصلاح دالة `handleBulkRenew` في `ContractsPage.tsx`

تغيير منطق حساب التواريخ (السطور 175-177):

**قبل:**
```
const today = new Date();
const startDate = today.toISOString().split('T')[0];
const endDate = new Date(today.getFullYear() + 1, ...).toISOString().split('T')[0];
```

**بعد:**
```
// لكل عقد: البداية = تاريخ انتهاء العقد القديم
// المدة = نفس مدة العقد الأصلي (الفرق بين start_date و end_date)
const oldStart = new Date(contract.start_date);
const oldEnd = new Date(contract.end_date);
const durationMs = oldEnd.getTime() - oldStart.getTime();

const newStart = oldEnd; // يبدأ من تاريخ انتهاء القديم
const newEnd = new Date(newStart.getTime() + durationMs);

const startDate = newStart.toISOString().split('T')[0];
const endDate = newEnd.toISOString().split('T')[0];
```

هذا يضمن:
- العقد الجديد يبدأ فوراً عند انتهاء القديم (بدون فجوة)
- مدة العقد الجديد مطابقة لمدة العقد القديم
- لا يتأثر بتاريخ يوم التجديد
