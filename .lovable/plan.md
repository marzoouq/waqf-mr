

# خطة التفصيل الدقيق للمصروفات والإيرادات

## الوضع الحالي
- صفحة التقارير (`ReportsPage.tsx`): تعرض "إجمالي المصروفات" و"إجمالي الإيرادات" كأرقام مجملة فقط
- ملف PDF: نفس المشكلة -- سطر واحد للمصروفات وسطر واحد للإيرادات
- صفحة إفصاح المستفيد: مفصلة بالفعل (لا تحتاج تعديل)

## التعديلات المطلوبة

### 1. صفحة التقارير (`ReportsPage.tsx`) -- جدول الإفصاح السنوي

**قسم الإيرادات المفصل:**
- عنوان فرعي "الإيرادات" بلون أخضر
- سطر لكل مصدر دخل (إيجارات، أخرى...) بمسافة بادئة
- سطر إجمالي الإيرادات بخلفية مميزة

**قسم المصروفات المفصل:**
- عنوان فرعي "المصروفات" بلون أحمر
- سطر لكل نوع مصروف (كهرباء، مياه، صيانة، عمالة، منصة إيجار، كتابة عقود، تأمين...) بمسافة بادئة
- سطر إجمالي المصروفات بخلفية مميزة

**قسم صافي الريع والتوزيع:**
- صافي الريع
- حصة الناظر (10%)
- حصة الواقف (5%)
- ريع المستفيدين

**إضافة جدول تفصيل المصروفات حسب العقار:**
- جدول جديد يوضح كم صُرف على كل عقار مع تفصيل الأنواع

### 2. ملف PDF (`pdfGenerator.ts`)

**تحديث واجهة `ReportData`:**
- إضافة `expensesByType: Array<{ type: string; amount: number }>`
- إضافة `incomeBySource: Array<{ source: string; amount: number }>`

**تعديل جدول الملخص في PDF:**
- استبدال سطر "إجمالي الإيرادات" الواحد بأسطر مفصلة لكل مصدر + سطر إجمالي
- استبدال سطر "إجمالي المصروفات" الواحد بأسطر مفصلة لكل نوع + سطر إجمالي
- إبقاء أسطر صافي الريع والحصص كما هي

### 3. تحديث استدعاء PDF في `ReportsPage.tsx`

- تمرير `expensesByType` و `incomeBySource` كبيانات إضافية لدالة `generateAnnualReportPDF`

### تعديل `ReportData` في `pdfGenerator.ts`:
```text
interface ReportData {
  fiscalYear: string;
  totalIncome: number;
  totalExpenses: number;
  netRevenue: number;
  adminShare: number;
  waqifShare: number;
  waqfRevenue: number;
  expensesByType: Array<{ type: string; amount: number }>;
  incomeBySource: Array<{ source: string; amount: number }>;
  beneficiaries: Array<{ name: string; percentage: number; amount: number }>;
}
```
