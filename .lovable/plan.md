

# اضافة تحميل فاتورة PDF وربط التسديد بالتحصيل التلقائي

## الهدف
1. زر تحميل فاتورة PDF لكل دفعة مستحقة يحتوي على بيانات العقد والمستأجر والمبلغ وتاريخ الاستحقاق
2. ربط تسديد الفاتورة بتحديث `tenant_payments` وتسجيل الدخل تلقائيا

---

## التغييرات المطلوبة

### 1. انشاء دالة توليد PDF للفاتورة (ملف جديد)
**ملف:** `src/utils/pdf/paymentInvoice.ts`

- دالة `generatePaymentInvoicePDF` تستقبل بيانات الفاتورة (رقم الفاتورة، اسم المستأجر، رقم العقد، العقار، المبلغ، تاريخ الاستحقاق، رقم الدفعة، الحالة) + معلومات الوقف
- تستخدم نفس البنية الموجودة في `core.ts` (خطوط عربية Amiri، هيدر رسمي، اطار مزخرف، فوتر)
- تصميم الفاتورة:
  - هيدر الوقف الرسمي (الشعار + الاسم + الصك)
  - عنوان "فاتورة دفعة مستحقة"
  - جدول بيانات: رقم الفاتورة، رقم العقد، اسم المستأجر، العقار، رقم الدفعة، المبلغ، تاريخ الاستحقاق، الحالة
  - ملاحظة اسفل الجدول
  - فوتر مع التاريخ ورقم الصفحة

### 2. تحديث تصدير ملفات PDF
**ملف:** `src/utils/pdf/index.ts`

- اضافة تصدير الدالة الجديدة `generatePaymentInvoicePDF`

### 3. انشاء دالة RPC لتسديد الفاتورة مع تحديث التحصيل (migration)
**Migration جديد:**

دالة `pay_invoice_and_record_collection` تنفذ ذريا:
1. تحديث حالة الفاتورة الى `paid` مع تاريخ السداد والمبلغ
2. زيادة `paid_months` في `tenant_payments` بمقدار 1 (عبر `upsert_tenant_payment` او مباشرة)
3. انشاء سجل دخل في جدول `income` بالمبلغ المسدد
4. ارجاع النتيجة

دالة `unpay_invoice_and_revert_collection` للتراجع:
1. اعادة حالة الفاتورة الى `pending` او `overdue`
2. انقاص `paid_months` بمقدار 1
3. حذف سجل الدخل المرتبط

### 4. تحديث هوك فواتير الدفعات
**ملف:** `src/hooks/usePaymentInvoices.ts`

- تحديث `useMarkInvoicePaid` لاستخدام دالة RPC الجديدة بدلا من التحديث المباشر
- تحديث `useMarkInvoiceUnpaid` لاستخدام دالة RPC العكسية
- اضافة `invalidateQueries` لـ `tenant_payments` و `income` عند النجاح

### 5. تحديث واجهة فواتير الدفعات
**ملف:** `src/components/contracts/PaymentInvoicesTab.tsx`

- اضافة زر تحميل PDF (ايقونة `Download`) بجانب كل فاتورة في الجدول وبطاقات الموبايل
- استدعاء `generatePaymentInvoicePDF` عند الضغط مع تمرير بيانات الفاتورة ومعلومات الوقف
- اضافة `usePdfWaqfInfo` للحصول على بيانات الوقف

## تفاصيل فنية

### تصميم الفاتورة PDF
```text
+----------------------------------+
|      [شعار الوقف]                |
|      اسم الوقف                   |
|      صك رقم: XXX - المحكمة       |
|  ________________________________|
|                                  |
|     فاتورة دفعة مستحقة          |
|                                  |
|  رقم الفاتورة:  XXX-INV-01      |
|  رقم العقد:     XXX             |
|  المستأجر:      عبدالرحمن        |
|  العقار:        101              |
|  رقم الدفعة:    3 من 12          |
|  المبلغ:        1,350 ر.س        |
|  تاريخ الاستحقاق: 2025-07-30    |
|  الحالة:        قيد الانتظار      |
|                                  |
|  ________________________________|
|  التاريخ    اسم الوقف    1/1     |
+----------------------------------+
```

### دالة RPC `pay_invoice_and_record_collection`
- تستقبل: `p_invoice_id`, `p_paid_amount`
- تجلب بيانات الفاتورة والعقد
- تحدث الفاتورة (status=paid, paid_date, paid_amount)
- تستدعي `upsert_tenant_payment` لزيادة paid_months + انشاء سجل دخل
- ترجع jsonb بالنتيجة

### دالة RPC `unpay_invoice_and_revert_collection`
- تستقبل: `p_invoice_id`
- تعيد الفاتورة لحالة pending/overdue
- تنقص paid_months بمقدار 1
- تحذف سجل الدخل المرتبط (بالبحث عن contract_id + تاريخ اليوم + المبلغ)
