

# إصلاح تسجيل الدخول برقم الهوية الوطنية

## المشكلة
صفحة تسجيل الدخول تبحث في جدول المستفيدين مباشرة عن رقم الهوية، لكن المستخدم لم يسجل دخوله بعد، فسياسات الأمان تمنعه من قراءة الجدول ويظهر له "رقم الهوية غير مسجل".

## الحل
إنشاء وظيفة خلفية جديدة تبحث عن البريد الإلكتروني المرتبط برقم الهوية بصلاحيات الخادم.

## الملفات المتأثرة

### 1. إنشاء `supabase/functions/lookup-national-id/index.ts`
- تستقبل `national_id` في الطلب
- تبحث في جدول `beneficiaries` باستخدام مفتاح الخدمة (يتجاوز RLS)
- ترجع البريد الإلكتروني فقط

### 2. تعديل `supabase/config.toml`
- إضافة إعداد الوظيفة الجديدة مع `verify_jwt = false`

### 3. تعديل `src/pages/Auth.tsx`
- استبدال الاستعلام المباشر لقاعدة البيانات (سطر 60-64) باستدعاء الوظيفة الخلفية الجديدة:

```text
// قبل (لا يعمل بسبب RLS):
supabase.from('beneficiaries').select('email').eq('national_id', nationalId)

// بعد:
supabase.functions.invoke('lookup-national-id', { body: { national_id: nationalId } })
```

## التفاصيل التقنية

### وظيفة `lookup-national-id`
- تستخدم `SUPABASE_SERVICE_ROLE_KEY` لإنشاء عميل Supabase بصلاحيات كاملة
- ترجع فقط `{ email: "..." }` بدون أي بيانات حساسة أخرى
- تتعامل مع CORS بشكل صحيح
- لا تحتاج مصادقة (لأنها تُستدعى قبل تسجيل الدخول)
