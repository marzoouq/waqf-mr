

# تقرير الفحص الجنائي الشامل للوحة المستفيدين

## نتيجة الفحص: تم اكتشاف 7 مشكلات (1 حرجة، 3 متوسطة، 3 طفيفة)

---

## 1. مشكلة حرجة: عدم فلترة العقود حسب السنة المالية

**الملف:** `ContractsViewPage.tsx` (سطر 21)

**الوصف:** صفحة العقود تستخدم `useContracts()` الذي يجلب **جميع العقود من كل السنوات** بدون أي فلتر، بينما صفحة الإفصاح وصفحة الحسابات الختامية تستخدم `useContractsByFiscalYear(fiscalYearId)` بشكل صحيح.

**الأثر:** يرى المستفيد عقود سنوات مقفلة مختلطة مع السنة الحالية.

**الحل:** استبدال `useContracts()` بـ `useContractsByFiscalYear(fiscalYearId)` مع استيراد `useFiscalYear`.

---

## 2. مشكلة متوسطة: صفحة الحسابات الختامية تفلتر العقود يدويا بدل استخدام الهوك الجاهز

**الملف:** `AccountsViewPage.tsx` (سطر 4 و48-52)

**الوصف:** تستورد `useContracts` (بدون فلتر) ثم تفلتر يدويا في `useMemo`:
```typescript
const { data: allContracts = [] } = useContracts();
const contracts = useMemo(() => {
  if (!fiscalYearId || fiscalYearId === 'all') return allContracts;
  return allContracts.filter(c => c.fiscal_year_id === fiscalYearId);
}, [allContracts, fiscalYearId]);
```
هذا يجلب **جميع العقود** من الخادم ثم يفلترها محليا، بينما `useContractsByFiscalYear` يفلتر من الخادم مباشرة.

**الأثر:** استهلاك بيانات وذاكرة غير ضروري + تكرار منطق الفلترة.

**الحل:** استبدال `useContracts` بـ `useContractsByFiscalYear(fiscalYearId)` كما في `DisclosurePage.tsx`.

---

## 3. مشكلة متوسطة: عدم دعم "جميع السنوات" في صفحتي العقارات والعقود

**الملف:** `DashboardLayout.tsx` (سطر 105-116)

**الوصف:** مسارات `/beneficiary/properties` و `/beneficiary/contracts` غير مضافة لقائمة `SHOW_ALL_ROUTES`، لذا لا يظهر خيار "جميع السنوات" في منتقي السنة المالية لهاتين الصفحتين.

**الأثر:** لا يستطيع المستفيد عرض عقود جميع السنوات من صفحة العقود المخصصة.

**الحل:** اضافة `/beneficiary/contracts` و `/beneficiary/properties` لمصفوفة `SHOW_ALL_ROUTES`.

---

## 4. مشكلة متوسطة: خطأ React key في صفحة العقارات (Fragment بدون key)

**الملف:** `PropertiesViewPage.tsx` (سطر 142 و 157)

**الوصف:** في عرض الجدول، يُستخدم `<>...</>` (Fragment) لتجميع صف العقار وصف الوحدات الفرعي، لكن React يتطلب key فريد على الـ Fragment عند الاستخدام داخل `.map()`. الكود الحالي يضع key على `TableRow` الداخلي لكن ليس على Fragment.

**الأثر:** تحذيرات React في الكونسول وسلوك غير متوقع عند اعادة الترتيب.

**الحل:** استخدام `<React.Fragment key={prop.id}>` بدلا من `<>`.

---

## 5. مشكلة طفيفة: عدم وجود حالة "فارغة" في صفحة العقود

**الملف:** `ContractsViewPage.tsx`

**الوصف:** عند عدم وجود عقود، لا تظهر رسالة واضحة. الجدول يكون فارغا بدون أي توضيح للمستفيد.

**الحل:** اضافة رسالة "لا توجد عقود مسجلة" مع ايقونة كما في الصفحات الاخرى.

---

## 6. مشكلة طفيفة: عدم وجود حالة "خطأ" في صفحتي العقارات والعقود

**الملفان:** `ContractsViewPage.tsx` و `PropertiesViewPage.tsx`

**الوصف:** جميع صفحات المستفيد الاخرى (الرئيسية، حصتي، الافصاح، التقارير، الفواتير، المراسلات، الاشعارات، اللائحة، الاعدادات) تتعامل مع حالة `isError` وتعرض واجهة خطأ مع زر "اعادة المحاولة". بينما صفحتا العقارات والعقود لا تتعاملان مع الاخطاء.

**الحل:** اضافة حالة `isError` مع واجهة خطأ متسقة مع باقي الصفحات.

---

## 7. مشكلة طفيفة: غياب اختبارات الوحدة لصفحتي العقارات والعقود

**الملفان:** لا يوجد `ContractsViewPage.test.tsx` ولا `PropertiesViewPage.test.tsx`

**الوصف:** جميع صفحات المستفيد الاخرى لديها ملفات اختبار مقابلة. الصفحتان الجديدتان فقط بدون اختبارات.
