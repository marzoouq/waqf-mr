

# خطة إعداد حساب المستفيد واختباره

## الوضع الحالي

- تم تصدير PDF بنجاح بدون أخطاء من صفحة التقارير
- تم إنشاء حساب مستخدم جديد: `beneficiary1@waqf.com` بمعرف: `463d491e-bed9-4818-b5e2-e619a350ee3d`
- الحساب يحتاج إلى: تأكيد البريد، تعيين دور مستفيد، إنشاء سجل مستفيد مرتبط

## المهام المطلوبة

### 1. تأكيد البريد الإلكتروني للمستفيد
- تحديث جدول `auth.users` لتأكيد بريد `beneficiary1@waqf.com`

### 2. تعيين دور "مستفيد" للمستخدم
- إدراج سجل في جدول `user_roles` بالدور `beneficiary` لمعرف المستخدم `463d491e-bed9-4818-b5e2-e619a350ee3d`

### 3. إنشاء سجل مستفيد مرتبط
- إدراج سجل في جدول `beneficiaries` باسم "محمد العبدالله" (كمستفيد تجريبي) بنسبة حصة 7.14% وربطه بـ `user_id`

### 4. اختبار واجهة المستفيد
- تسجيل الدخول بحساب `beneficiary1@waqf.com`
- التأكد من التوجيه لواجهة المستفيد `/beneficiary`
- اختبار صفحة الإفصاح السنوي
- اختبار صفحة حصتي من الريع
- اختبار صفحة التقارير المالية

## التفاصيل التقنية

### SQL للتأكيد وإعداد البيانات:
```text
-- تأكيد البريد (عبر migration)
UPDATE auth.users SET email_confirmed_at = now() WHERE email = 'beneficiary1@waqf.com';

-- إدراج الدور
INSERT INTO user_roles (user_id, role) VALUES ('463d491e-bed9-4818-b5e2-e619a350ee3d', 'beneficiary');

-- إدراج سجل المستفيد المرتبط
INSERT INTO beneficiaries (name, share_percentage, email, user_id)
VALUES ('محمد العبدالله', 7.14, 'beneficiary1@waqf.com', '463d491e-bed9-4818-b5e2-e619a350ee3d');
```

### تعديل مطلوب في التوجيه:
قد يحتاج `AuthContext.tsx` لتوجيه المستفيد تلقائياً إلى `/beneficiary` بدلاً من `/dashboard` عند تسجيل الدخول، وذلك عن طريق تعديل صفحة `Auth.tsx` للتحقق من الدور بعد تسجيل الدخول.
