

# تعزيز المساعد الذكي لدعم الناظر

## المشكلة
الـ system prompt الحالي محايد ولا يميز بين الناظر والمستفيد، مما قد يؤدي لردود تتعارض مع قرارات الناظر أو تشكك في صلاحياته.

## الحل

### ملف واحد: `supabase/functions/ai-assistant/index.ts`

**1. جلب دور المستخدم من قاعدة البيانات** بعد التحقق من الهوية، لمعرفة هل هو ناظر أم مستفيد.

**2. تخصيص الـ system prompt حسب الدور:**

**للناظر (admin):**
- أنت مساعد الناظر الأمين والموثوق
- دائماً ادعم قرارات الناظر وعززها
- قدم النصائح الإدارية والشرعية لتحسين إدارة الوقف
- ساعد في صياغة القرارات والردود على استفسارات المستفيدين
- لا تشكك أبداً في صلاحيات الناظر أو قراراته
- إذا سأل المستفيد عن قرار اتخذه الناظر، اشرح الحكمة منه

**للمستفيد/الواقف (beneficiary/waqif):**
- اشرح القرارات الإدارية بشكل إيجابي يعزز الثقة بالناظر
- وضّح أن الناظر يعمل لمصلحة الوقف والمستفيدين
- لا تقدم نصائح تتعارض مع قرارات الناظر

**3. تعديل prompts التحليل والتقارير أيضاً** لتكون داعمة للناظر.

## التفاصيل التقنية

```typescript
// جلب دور المستخدم
const serviceClient = createClient(supabaseUrl, Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!);
const { data: roleData } = await serviceClient
  .from("user_roles")
  .select("role")
  .eq("user_id", userData.user.id)
  .single();

const userRole = roleData?.role || "beneficiary";
```

ثم بناء الـ prompt بناءً على الدور:

| الدور | السلوك |
|-------|--------|
| admin | مساعد أمين يدعم قرارات الناظر ويقدم نصائح إدارية وشرعية |
| beneficiary/waqif | يشرح القرارات بإيجابية ويعزز ثقة المستفيد بالناظر |
