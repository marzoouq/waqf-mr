
# اصلاح مشكلة تعلّق زر تسجيل الدخول

## تشخيص المشكلة الجذرية

بعد تتبع تدفق الكود خطوة بخطوة:

1. المستخدم يضغط "تسجيل الدخول" -> `setIsLoading(true)` -> الزر يتحول لـ "جاري تسجيل الدخول..."
2. `signIn` ينجح -> `finally` يعمل -> `setIsLoading(false)` -> الزر يجب أن يعود طبيعياً
3. لكن `onAuthStateChange` في `AuthContext` يستدعي `fetchUserRole` مع إعادة محاولات (500ms + 1000ms)
4. خلال هذه الفترة: `user` موجود، `role = null`، `loading = false`
5. المستخدم يبقى على صفحة `/auth` ينتظر التوجيه

**المشكلة الحقيقية**: حتى لو الزر تقنياً يعود لحالته، المستخدم لا يُوجَّه لأن `role` لم يُحدد، والصفحة تبقى معروضة بدون أي مؤشر على التقدم. السيناريو الأسوأ: إذا فشل `fetchUserRole` بعد كل المحاولات، يبقى `role = null` ومؤقت الحماية (4 ثوان) يوجه لـ `/beneficiary` -- لكن هذا قد يفشل أيضاً إذا كان الدور الفعلي هو `admin`.

## الإصلاحات المقترحة

### 1. إضافة مؤقت أمان صارم للزر (Auth.tsx)

إضافة `useEffect` يفرض إعادة `isLoading` إلى `false` بعد 10 ثوانٍ كحد أقصى مهما حدث:

```typescript
useEffect(() => {
  if (!isLoading) return;
  const safety = setTimeout(() => {
    setIsLoading(false);
    console.warn('[Auth] Force-reset isLoading after 10s safety timeout');
  }, 10000);
  return () => clearTimeout(safety);
}, [isLoading]);
```

### 2. إظهار حالة "جاري التحقق" بعد نجاح تسجيل الدخول (Auth.tsx)

بدلاً من ترك الزر في حالة غامضة بعد نجاح المصادقة، إضافة حالة مرئية واضحة:

```typescript
// إذا المستخدم مسجّل لكن ينتظر التوجيه، عرض شاشة تحميل
if (user && !loading) {
  return (
    <div className="min-h-screen flex items-center justify-center">
      <Loader2 className="w-8 h-8 animate-spin text-primary" />
      <span>جاري التحقق من الصلاحيات...</span>
    </div>
  );
}
```

هذا يضمن أنه بمجرد نجاح تسجيل الدخول (user موجود)، تختفي صفحة الدخول بالكامل ويظهر مؤشر تحميل واضح بدلاً من الزر المعلّق.

### 3. إضافة سجلات تتبع تشخيصية (Auth.tsx)

إضافة `console.log` في نقاط حرجة لتشخيص المشكلة إذا تكررت:

- قبل وبعد `signIn`
- في `finally`
- في useEffect التوجيه

### 4. تحسين مؤقت الحماية للتوجيه (Auth.tsx)

تقليل مؤقت الحماية من 4 ثوانٍ إلى 3 ثوانٍ، وإعادة جلب الدور مرة أخيرة قبل التوجيه الاحتياطي.

## النتيجة المتوقعة

- بمجرد نجاح المصادقة: تختفي صفحة الدخول فوراً ويظهر "جاري التحقق من الصلاحيات..."
- بعد تحديد الدور: توجيه تلقائي للوحة المناسبة
- إذا فشل تحديد الدور: توجيه احتياطي بعد 3 ثوانٍ
- الزر لا يمكن أن يعلق أبداً (مؤقت أمان 10 ثوانٍ كحد أقصى)
