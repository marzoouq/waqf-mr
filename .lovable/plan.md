

# إصلاح الطباعة وإضافة خيارات الدفع في العقود

## المشاكل المكتشفة

### 1. الطباعة تظهر فارغة
مربع حوار الوحدات يفتح داخل `Dialog` من Radix. عند استدعاء `window.print()`، القاعدة CSS التالية تخفي الحاوية الرئيسية للحوار:
```css
.fixed:not([role="dialog"]) { display: none !important; }
```
هذه القاعدة تخفي عنصر `[data-radix-dialog-overlay]` الأب الذي يحتوي على الحوار نفسه، فيظهر المحتوى فارغاً.

### 2. لا توجد خيارات دفع في نموذج العقود
نموذج إضافة العقد في `ContractsPage.tsx` يحتوي فقط على حقل `rent_amount` واحد بدون:
- نوع الدفع (سنوي / شهري / متعدد)
- عدد الدفعات
- قيمة الدفعة الواحدة

جدول `contracts` في قاعدة البيانات لا يحتوي أعمدة لهذه البيانات.

## ما سيتم تنفيذه

### 1. إصلاح الطباعة (ملف `src/index.css`)

تعديل قواعد `@media print` لضمان ظهور محتوى الحوار:
- إضافة قاعدة لإظهار الحاوية الثابتة التي تحتوي على `[role="dialog"]` بدلاً من إخفائها
- تعديل القاعدة `.fixed:not([role="dialog"])` لاستثناء الحاوية الأب للحوار
- إضافة `[data-radix-portal]` كعنصر مرئي عند الطباعة

### 2. إضافة أعمدة الدفع في قاعدة البيانات (migration)

إضافة 3 أعمدة جديدة لجدول `contracts`:
- `payment_type` (text, default: 'annual') - نوع الدفع: annual / monthly / multi
- `payment_count` (integer, default: 1) - عدد الدفعات
- `payment_amount` (numeric, nullable) - قيمة الدفعة الواحدة (تحسب تلقائياً)

### 3. تعديل نموذج العقود (ملف `src/pages/dashboard/ContractsPage.tsx`)

إضافة الحقول التالية في نموذج إضافة/تعديل العقد:
- **قائمة منسدلة "نوع الدفع"**: سنوي / شهري / دفعات متعددة
- **حقل "عدد الدفعات"**: يظهر فقط عند اختيار "دفعات متعددة" (من 2 إلى 12)
- **عرض تلقائي لقيمة الدفعة الواحدة**: `rent_amount / payment_count`

المنطق:
- سنوي: عدد الدفعات = 1، قيمة الدفعة = كامل الإيجار
- شهري: عدد الدفعات = 12، قيمة الدفعة = الإيجار / 12
- متعدد: المستخدم يحدد العدد يدوياً

### 4. تحديث نوع البيانات (ملف `src/types/database.ts`)

إضافة الحقول الجديدة لواجهة `Contract`:
- `payment_type?: string`
- `payment_count?: number`
- `payment_amount?: number`
