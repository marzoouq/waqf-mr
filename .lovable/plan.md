# تقييد الإفصاح والبيانات المالية للمستفيد حسب السنة المنشورة

## المشكلة الحالية

توجد ثغرة تسمح للمستفيد برؤية بيانات سنوات مالية محجوبة (غير منشورة) بعدة طرق:

1. خيار "جميع السنوات" في القائمة المنسدلة يتجاوز فلترة السنة المالية ويعرض جميع البيانات
2. جداول العقود والدخل والمصروفات والفواتير لا تحتوي سياسات RLS تمنع الوصول لبيانات سنوات غير منشورة
3. عند عدم وجود سنوات منشورة، النظام يعود لوضع "all" ويعرض كل البيانات

## الحل المقترح (طبقتان من الحماية)

### الطبقة الاولى: حماية قاعدة البيانات (RLS)

اضافة سياسات RLS تقييدية (RESTRICTIVE) على 4 جداول لمنع المستفيد والواقف من رؤية بيانات السنوات المحجوبة:

- **contracts**: منع SELECT اذا كانت السنة المالية المرتبطة غير منشورة
- **income**: نفس القيد
- **expenses**: نفس القيد
- **invoices**: نفس القيد

```text
السياسة: اذا كان المستخدم مستفيد او واقف:
  - اذا fiscal_year_id = NULL -> السماح (بيانات غير مرتبطة بسنة)
  - اذا fiscal_year_id موجود -> يجب ان تكون السنة published = true
  - اذا كان المستخدم admin او accountant -> السماح دائما
```

### الطبقة الثانية: حماية واجهة المستخدم

**تعديل `FiscalYearContext.tsx`:**

- عند عدم وجود سنوات مالية متاحة (كلها محجوبة)، عدم الرجوع لـ `'all'` بل البقاء بدون بيانات
- اضافة حالة `noPublishedYears` لعرض رسالة مناسبة للمستفيد

**تعديل `DashboardLayout.tsx`:**

- ازالة صفحات المستفيد من قائمة `SHOW_ALL_ROUTES` لمنع ظهور خيار "جميع السنوات"
- الصفحات المتاثرة: الافصاح، حصتي، التقارير، العقود، العقارات، الفواتير

**تعديل صفحات المستفيد:**

- اضافة رسالة تنبيهية عند عدم وجود سنوات منشورة: "لم ينشر الناظر اي سنة مالية بعد"

## التفاصيل التقنية

### 1. ترحيل قاعدة البيانات (Migration)

انشاء دالة مساعدة `is_fiscal_year_published` واضافة سياسات RESTRICTIVE:

```sql
-- دالة للتحقق من نشر السنة المالية
CREATE OR REPLACE FUNCTION public.is_fiscal_year_accessible(p_fiscal_year_id uuid)
RETURNS boolean
LANGUAGE sql STABLE SECURITY DEFINER SET search_path = public
AS $$
  SELECT CASE
    WHEN has_role(auth.uid(), 'admin') OR has_role(auth.uid(), 'accountant') THEN true
    WHEN p_fiscal_year_id IS NULL THEN true
    ELSE EXISTS (
      SELECT 1 FROM public.fiscal_years
      WHERE id = p_fiscal_year_id AND published = true
    )
  END;
$$;

-- سياسات تقييدية على contracts, income, expenses, invoices
-- (تطبق بالتوازي مع السياسات الموجودة)
```

### 3. ملاحظات

- **العقارات**: تبقى مرئية دائما كاصول ثابتة (لا تملك fiscal_year_id). المؤشرات المالية فيها ستتقيد تلقائيا لان العقود والمصروفات ستكون مقيدة
- لا تاثير على الناظر والمحاسب - يرون كل شيء كالمعتاد
- الحماية مزدوجة: حتى لو تم تجاوز الواجهة، قاعدة البيانات تمنع الوصول
- المستفيد لا يرى الا ما ينشر ويفتح من الناظر 
- مراجعة لوحة تحكم الناظر وصلاحيات على لوحة المستفيدين
