# دليل المساهمة وبروتوكول التنفيذ الرسمي

هذا المستودع يعتمد بروتوكول تنفيذ إلزامي قبل أي دفع إلى الفرع الرئيسي.

## الهدف

- منع الإصلاحات الشكلية غير المتحققة
- فرض تحقق موضوعي قبل الدمج أو الدفع
- توحيد جودة التنفيذ بين جميع المساهمين

## نطاق التطبيق

- أي تغيير في الكود
- أي تغيير في قاعدة البيانات أو `supabase/migrations`
- أي تغيير أمني أو متعلق بالصلاحيات أو RLS

## بوابات إلزامية قبل أي Push

### 1. Baseline Check

نفّذ التحقق التالي قبل بدء الإصلاح:

```bash
git status --short
git branch --show-current
git log --oneline -5
```

المطلوب:

- توثيق حالة الشجرة قبل التغيير
- عدم تجاهل تغييرات غير متوقعة

### 2. Root Cause Check

يمنع تنفيذ تعديل دون تحديد السبب الجذري كتابةً.

المطلوب:

- وصف المشكلة الفعلية
- ربط السبب بالملفات أو السياسات أو الهجرات ذات الصلة

### 3. Minimal Change Rule

المطلوب:

- أقل سطح تعديل ممكن
- منع تعديل ملفات غير مرتبطة بالمشكلة
- منع حذف كود قائم بلا مبرر واضح

### 4. Security & Data Gate

إذا كان التغيير يخص الأمن أو البيانات:

- مراجعة RLS والسياسات ذات الصلة
- مراجعة grants/revokes للعروض والجداول
- مراجعة Edge Functions للتحقق من `getUser()`
- منع أي اعتماد على `getSession()` داخل Edge Functions

### 5. Verification Matrix

نفّذ التحقق التالي قبل الدفع:

```bash
npm run lint
npx tsc --noEmit
npm run build
npm test
npm audit
```

المطلوب:

- نجاح lint
- نجاح TypeScript
- نجاح build
- نجاح الاختبارات
- عدم وجود ثغرات مفتوحة تتطلب معالجة فورية

### 6. Migration Safety Gate

إذا كان التغيير يشمل `supabase/migrations`:

- تأكيد ترتيب الهجرة زمنيًا
- منع كسر التوافق مع الهجرات السابقة
- توثيق سبب كل `DROP/CREATE` في العروض والسياسات
- التحقق من صلاحيات `anon` و`authenticated` و`service_role`

### 7. Pre-Push Checklist

يجب إكمال البنود التالية قبل الدفع:

- تم تحديد السبب الجذري
- تم تنفيذ أقل تعديل كافٍ
- تم اجتياز مصفوفة التحقق كاملة
- لا توجد تغييرات غير مقصودة خارج نطاق المهمة
- رسالة commit واضحة وتصف السبب والأثر

## قوالب أوامر تشغيل سريعة

### فحص سريع قبل commit

```bash
git status --short
npm run lint
```

### فحص كامل قبل push

```bash
npm run lint && npx tsc --noEmit && npm run build && npm test && npm audit
```

## معايير الرفض

يُرفض الدفع إذا تحقق أي شرط مما يلي:

- فشل أي خطوة من مصفوفة التحقق
- غياب تحليل السبب الجذري
- تغييرات غير مرتبطة بالمشكلة
- تعديل أمني بدون تحقق RLS/grants
