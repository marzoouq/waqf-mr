## 1. لوحة التحكم (`AdminDashboard`)

### الغرض
عرض ملخص شامل لحالة الوقف المالية والتشغيلية للسنة المالية المختارة.

### مصادر البيانات
```
useActiveFiscalYear()  → السنة المالية النشطة + قائمة السنوات
useProperties()        → عدد العقارات
useContracts()         → العقود (نشطة/منتهية)
useAllUnits()          → الوحدات (لحساب الإشغال)
useFinancialSummary()  → الملخص المالي الموحد
```

### البطاقات الإحصائية (9 بطاقات)
| البطاقة | المصدر |
|---------|--------|
| إجمالي العقارات | `properties.length` |
| العقود النشطة | `contracts.filter(status === 'active')` |
| الإيرادات التعاقدية | `Σ contracts.rent_amount` |
| إجمالي الدخل الفعلي | `totalIncome` من useFinancialSummary |
| إجمالي المصروفات | `totalExpenses` من useFinancialSummary |
| حصة الناظر | `adminShare` (محسوبة ديناميكياً) |
| حصة الواقف | `waqifShare` (محسوبة ديناميكياً) |
| ريع الوقف للمستفيدين | `waqfRevenue` |
| عدد المستفيدين | `beneficiaries.length` |

### مؤشرات الأداء الرئيسية (KPI)
```
نسبة التحصيل   = (الدخل الفعلي / الإيرادات التعاقدية) × 100
معدل الإشغال    = (الوحدات المؤجرة / إجمالي الوحدات) × 100
متوسط الإيجار   = الإيرادات التعاقدية / عدد العقود النشطة
نسبة المصروفات  = (المصروفات / الدخل الفعلي) × 100
```

**الترميز اللوني:**
- أخضر: ≥ 80% (تحصيل/إشغال) أو ≤ 20% (مصروفات)
- أصفر: 50-79% (تحصيل/إشغال) أو 21-40% (مصروفات)
- أحمر: < 50% (تحصيل/إشغال) أو > 40% (مصروفات)

### الرسوم البيانية
1. **بياني شريطي** — الدخل والمصروفات الشهرية (تجميع حسب `date.substring(0,7)`)
2. **بياني دائري** — توزيع المصروفات حسب النوع (`expense_type`)

## 2. الحسابات الختامية (`AccountsPage`)

### الغرض
الصفحة المركزية لإدارة التسلسل المالي الهرمي، إدخال القيم اليدوية، متابعة التحصيل، وإقفال السنة المالية.

### مصادر البيانات
```
useAccounts()              → سجلات الحسابات المحفوظة
useIncomeByFiscalYear()    → الإيرادات مفلترة بالسنة
useExpensesByFiscalYear()  → المصروفات مفلترة بالسنة
useContracts()             → العقود لحساب التحصيل
useBeneficiaries()         → المستفيدون وحصصهم
useTenantPayments()        → سجل دفعات المستأجرين
useAppSettings()           → نسب الناظر/الواقف والإعدادات
```

### التسلسل المالي الهرمي

```
┌─────────────────────────────────────────────────────────────┐
│  إيرادات العام الحالي (totalIncome)                         │
│  + رقبة الوقف المرحلة (waqfCorpusPrevious)                  │
│  ══════════════════════════════════════                      │
│  = الإجمالي الشامل (grandTotal)                             │
│  - المصروفات التشغيلية (totalExpenses)                       │
│  ══════════════════════════════════════                      │
│  = الصافي بعد المصاريف (netAfterExpenses)                   │
│  - ضريبة القيمة المضافة (manualVat) ← إدخال يدوي           │
│  ══════════════════════════════════════                      │
│  = الصافي بعد الضريبة (netAfterVat)                         │
│  - الزكاة (zakatAmount) ← إدخال يدوي                       │
│  ══════════════════════════════════════                      │
│  = الصافي بعد الزكاة (netAfterZakat)                        │
│                                                             │
│  أساس حساب الحصص (shareBase):                               │
│  = إيرادات العام - المصروفات - الزكاة                       │
│  ⚠️ يُستبعد: الرصيد المرحل + الضريبة                       │
│                                                             │
│  - حصة الناظر = shareBase × adminPercent%                   │
│  - حصة الواقف = shareBase × waqifPercent%                   │
│  ══════════════════════════════════════                      │
│  = ريع الوقف (waqfRevenue)                                  │
│  - رقبة الوقف للعام الحالي (waqfCorpusManual) ← يدوي       │
│  ══════════════════════════════════════                      │
│  = المبلغ المتاح (availableAmount)                          │
│  - التوزيعات (manualDistributions) ← يدوي                   │
│  ══════════════════════════════════════                      │
│  = الرصيد المتبقي (remainingBalance)                        │
└─────────────────────────────────────────────────────────────┘
```

### القيم اليدوية vs المحسوبة

| الحقل | النوع | المصدر |
|-------|-------|--------|
| `totalIncome` | محسوب | `Σ income.amount` |
| `totalExpenses` | محسوب | `Σ expenses.amount` |
| `waqfCorpusPrevious` | يدوي/مرحل | من السنة السابقة أو إدخال يدوي |
| `manualVat` | يدوي | إدخال الناظر |
| `zakatAmount` | يدوي | إدخال الناظر |
| `adminPercent` | إعداد | `app_settings['admin_share_percentage']` (افتراضي 10%) |
| `waqifPercent` | إعداد | `app_settings['waqif_share_percentage']` (افتراضي 5%) |
| `waqfCorpusManual` | يدوي | إدخال الناظر |
| `manualDistributions` | يدوي | إدخال الناظر |
| `adminShare` | محسوب | `shareBase × adminPercent / 100` |
| `waqifShare` | محسوب | `shareBase × waqifPercent / 100` |
| `waqfRevenue` | محسوب | `netAfterZakat - adminShare - waqifShare` |

### جدول التحصيل والمتأخرات

يتتبع النظام دفعات كل مستأجر عبر جدول `tenant_payments`:

```typescript
// حساب المتأخرات لكل عقد
expectedPayments = payment_type === 'monthly' ? 12 
                 : payment_type === 'multi'   ? payment_count 
                 : 1;

paymentPerPeriod = payment_amount || (rent_amount / payment_count);
totalCollected   = paymentPerPeriod × paidMonths;
arrears          = (paymentPerPeriod × expectedPayments) - totalCollected;
```

**التعديل المباشر (Inline Editing):**
- يستطيع الناظر تعديل اسم المستأجر، قيمة الدفعة، وعدد الدفعات المسددة
- عند الحفظ يتم تحديث `contracts` و `tenant_payments` معاً
- يُعاد حساب الإجماليات تلقائياً

### حفظ الحساب الختامي

عند الضغط على "حفظ الحساب":
1. يُنشأ سجل في `accounts` بكامل القيم المالية المحسوبة
2. يُرسل إشعار لجميع المستفيدين (`notify_all_beneficiaries`)
3. إذا كانت التوزيعات > 0، يُرسل إشعار إضافي بالتوزيعات

## 3. آلية إقفال السنة المالية والأرشفة

### المتطلبات المسبقة
- السنة المالية المختارة يجب أن تكون بحالة `active`
- إذا كانت `closed`، تظهر شارة "سنة مقفلة" ويُمنع التعديل

### خطوات الإقفال (`handleCloseYear`)

```
الخطوة (A): حفظ لقطة ختامية نهائية
├── إنشاء سجل accounts بكامل الأرقام الحالية
├── يضمن عدم فقدان أي بيانات

الخطوة (B): تغيير حالة السنة
├── UPDATE fiscal_years SET status = 'closed' WHERE id = selectedFY.id

الخطوة (C): إنشاء السنة المالية التالية
├── تاريخ البداية = تاريخ نهاية السنة المقفلة
├── تاريخ النهاية = +1 سنة
├── التسمية = "startYear-endYear" (مثال: "2025-2026")
├── يُتحقق أولاً من عدم وجود سنة مكررة

الخطوة (D): إنشاء حساب أولي للسنة الجديدة
├── waqf_corpus_previous = remainingBalance (الرصيد المتبقي)
├── جميع الحقول الأخرى = 0 (تصفير الحسابات التشغيلية)

الخطوة (E): تحديث الكاش
├── invalidateQueries(['fiscal_years'])
├── invalidateQueries(['accounts'])
├── invalidateQueries(['income'])
├── invalidateQueries(['expenses'])

الخطوة (F): إشعار المستفيدين
├── إشعار بالإقفال مع مبلغ الرصيد المرحل
```

### مخطط تدفق الإقفال

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  السنة       │     │  حفظ لقطة        │     │  تغيير الحالة    │
│  النشطة      │────▶│  ختامية (A)       │────▶│  إلى مغلقة (B)   │
└──────────────┘     └──────────────────┘     └────────┬─────────┘
                                                       │
                     ┌──────────────────┐              │
                     │  إشعار           │              │
                     │  المستفيدين (F)  │◀─────────────┤
                     └──────────────────┘              │
                                                       ▼
                     ┌──────────────────┐     ┌──────────────────┐
                     │  حساب أولي       │     │  إنشاء سنة       │
                     │  بالرصيد (D)     │◀────│  جديدة (C)       │
                     └──────────────────┘     └──────────────────┘
```

### حماية البيانات المقفلة

1. **على مستوى الواجهة**: عند `isClosed = true`، تُعطل أزرار التعديل والحفظ وتظهر أيقونة القفل
2. **على مستوى قاعدة البيانات**: الدالة `prevent_closed_fiscal_year_modification()` تمنع أي INSERT/UPDATE/DELETE على سجلات مرتبطة بسنة مقفلة
3. **تأكيد مزدوج**: يظهر `AlertDialog` قبل الإقفال مع تحذير بعدم إمكانية التراجع

## 4. التقارير (`ReportsPage`)

### الغرض
عرض تقارير مالية تفصيلية وتصديرها بصيغة PDF، مع مقارنة أداء العقارات.

### مصادر البيانات
```
useFinancialSummary()  → كافة البيانات المالية الموحدة
useProperties()        → قائمة العقارات
useContracts()         → العقود لحساب الإيرادات التعاقدية
useAllUnits()          → الوحدات لحساب الإشغال
```

### التبويبات

#### أ. التقارير المالية (`financial`)
- **الإفصاح السنوي**: جدول تفصيلي يعرض كامل التسلسل المالي الهرمي
- **توزيع الدخل**: بياني دائري حسب المصدر
- **توزيع المصروفات**: بياني دائري حسب النوع
- **توزيع حصص المستفيدين**: جدول + بياني شريطي

#### ب. مقارنة أداء العقارات (`performance`)
لكل عقار:
```typescript
annualRent     = Σ contracts.rent_amount (للعقار)
totalExpenses  = Σ expenses.amount (للعقار)
netIncome      = annualRent - totalExpenses
occupancy      = (وحدات مؤجرة / إجمالي الوحدات) × 100
```

### تصدير PDF

| الزر | الدالة | المحتوى |
|------|--------|---------|
| الإفصاح السنوي PDF | `generateAnnualDisclosurePDF()` | التسلسل المالي الكامل + حصص المستفيدين |
| تصدير التقرير | `generateAnnualReportPDF()` | ملخص مالي + توزيعات + مصروفات حسب النوع |

**خصائص PDF:**
- خط Amiri العربي
- اتجاه RTL
- ترويسة تحتوي على بيانات الوقف من `usePdfWaqfInfo()`
- فترة مالية مأخوذة من `currentAccount.fiscal_year`

## 5. العلاقة بين الصفحات الثلاث

```
AdminDashboard (عرض فقط)
    │
    │ useFinancialSummary() ←── مصدر بيانات مشترك
    │
    ▼
AccountsPage (تعديل + إقفال)
    │
    │ calculateFinancials() ←── منطق حسابي مشترك
    │                           (src/utils/accountsCalculations.ts)
    │
    │ handleCloseYear() ←── يُنشئ سجل ختامي + سنة جديدة
    │
    ▼
ReportsPage (عرض + تصدير)
    │
    │ useFinancialSummary() ←── يقرأ من السجل المحفوظ
    │                          (currentAccount) إذا كانت السنة مقفلة
    │
    ▼
  PDF Export ←── القراءة المباشرة من السجل المدقق
```

### مبدأ "القراءة من السجل المدقق"
- **السنة النشطة**: يتم الحساب ديناميكياً من بيانات الإيرادات والمصروفات الفعلية
- **السنة المقفلة**: يُقرأ من سجل `accounts` المحفوظ عند الإقفال (Snapshot)
- يضمن هذا النهج تطابقاً تاماً بين لوحة الناظر والتقارير والإفصاحات

## 7. الامتثال والمعايير

| المعيار | التطبيق |
|---------|---------|
| **SOCPA** | تسلسل مالي هرمي متوافق مع معايير الهيئة السعودية |
| **IFRS/GAAP** | فصل الفترات المالية، ترحيل الأرصدة، عدم تعديل السنوات المقفلة |
| **نظام الأوقاف** | حصص ناظر/واقف منفصلة، شفافية التوزيع |
| **Audit Trail** | دالة `audit_trigger_func()` تسجل كل عملية تعديل/حذف/إضافة |
| **RLS** | سياسات أمان صارمة على `accounts` و `fiscal_years` |

## 8. انتماء العقود للسنة المالية

### القاعدة
- كل عقد يُثبّت في السنة المالية التي أُنشئ فيها عبر حقل `fiscal_year_id`
- **العقود لا تنتقل بين السنوات المالية أبداً** (متوافق مع IFRS 16 و SOCPA)
- فقط المبلغ المرحل (`waqf_corpus_previous`) ينتقل بين السنوات عند الإقفال

### السلوك
| السيناريو | النتيجة |
|-----------|---------|
| إنشاء عقد جديد | يُربط تلقائياً بالسنة المالية النشطة |
| تعديل عقد | `fiscal_year_id` لا يتغير |
| إضافة سنة مالية أقدم | العقود تبقى في سنتها الأصلية |
