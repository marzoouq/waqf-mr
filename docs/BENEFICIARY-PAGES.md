# توثيق صفحات المستفيد والمنطق المالي

## التسلسل المالي الهرمي (Financial Hierarchy)

جميع صفحات المستفيد تعتمد على هوك موحد `useFinancialSummary` الذي يطبق التسلسل التالي:

```
┌─────────────────────────────────────────────────────────────────┐
│  إجمالي إيرادات العام الحالي (totalIncome)                      │
│  + رصيد رقبة الوقف المرحل من العام السابق (waqfCorpusPrevious)  │
│  ═══════════════════════════════════════════                     │
│  الإجمالي الشامل (grandTotal)                                   │
│  ─ المصروفات التشغيلية (totalExpenses)                          │
│  ═══════════════════════════════════════════                     │
│  صافي بعد المصروفات (netAfterExpenses)                          │
│  ─ ضريبة القيمة المضافة (vatAmount) ← إدخال يدوي               │
│  ═══════════════════════════════════════════                     │
│  صافي بعد الضريبة (netAfterVat)                                 │
│  ─ الزكاة (zakatAmount) ← إدخال يدوي                           │
│  ═══════════════════════════════════════════                     │
│  صافي بعد الزكاة (netAfterZakat)                                │
│                                                                 │
│  ┌─── أساس حساب الحصص (shareBase) ───────────────────────┐     │
│  │ = إيرادات العام فقط ─ المصروفات ─ الزكاة              │     │
│  │ (يستبعد: الرصيد المرحل + الضريبة)                     │     │
│  │ ─ حصة الناظر (adminShare) = shareBase × 10%           │     │
│  │ ─ حصة الواقف (waqifShare) = shareBase × 5%            │     │
│  └───────────────────────────────────────────────────────┘     │
│                                                                 │
│  ريع الوقف (waqfRevenue) = netAfterZakat ─ حصتا الناظر والواقف │
│  ─ استقطاع رقبة الوقف (waqfCorpusManual) ← إدخال يدوي         │
│  ═══════════════════════════════════════════                     │
│  المبلغ القابل للتوزيع (availableAmount)                        │
│  ─ التوزيعات الفعلية (distributionsAmount) ← إدخال يدوي        │
│  ═══════════════════════════════════════════                     │
│  الرصيد المتبقي (remainingBalance)                              │
└─────────────────────────────────────────────────────────────────┘
```

### حساب حصة المستفيد الفردي
```
حصة المستفيد = (ريع الوقف ─ استقطاع رقبة الوقف) × نسبة حصته ÷ مجموع النسب
             = availableAmount × share_percentage / totalBeneficiaryPercentage
```
> **ملاحظة:** يُستخدم `totalBeneficiaryPercentage` (مجموع نسب جميع المستفيدين) بدلاً من 100 لضمان التوزيع التناسبي الصحيح حتى لو لم يبلغ المجموع 100%.

### قواعد التوزيع الشرعية
- **الزوجات** مجتمعات: الثُمن (1/8 = 12.5%)
- **الأبناء والبنات**: الباقي بنسبة 2:1 (للذكر مثل حظ الأنثيين)
- **دقة الحصص**: 6 خانات عشرية لضمان العدالة في المبالغ الكبيرة
- **الرقابة**: مجموع النسب لا يتجاوز 100%

### مصدر النسب المالية
| النسبة | المفتاح في `app_settings` | القيمة الافتراضية |
|--------|---------------------------|-------------------|
| حصة الناظر | `admin_share_percentage` | 10% |
| حصة الواقف | `waqif_share_percentage` | 5% |

## الصفحات الأربع

### 1. صفحة الإفصاح السنوي (`DisclosurePage.tsx`)

**المسار**: `/beneficiary/disclosure`

**الدور**: المصدر الأساسي والوحيد للتسلسل المالي التفصيلي الكامل.

**البيانات المعروضة**:
- التسلسل المالي الهرمي كاملاً (من الإجمالي الشامل حتى الرصيد المتبقي)
- تفصيل الإيرادات حسب المصدر (`incomeBySource`)
- تفصيل المصروفات حسب النوع (`expensesByTypeExcludingVat`) — **تستبعد** ضريبة القيمة المضافة كونها بند مستقل
- حصة المستفيد المحسوبة ديناميكياً
- معلومات السنة المالية (الفترة من/إلى)

**المنطق الخاص**:
```typescript
// جلب بيانات المستفيد الحالي
const currentBeneficiary = beneficiaries.find(b => b.user_id === user?.id);

// حساب حصته من المبلغ القابل للتوزيع (تناسبي)
const myShare = currentBeneficiary 
  ? availableAmount * (currentBeneficiary.share_percentage / totalBeneficiaryPercentage) 
  : 0;
```

**مصدر القيم**: عند وجود حساب ختامي مخزن (`currentAccount`) تُقرأ القيم منه مباشرة (مبدأ القراءة من السجل المدقق). عند عدم وجوده تُحسب ديناميكياً.

### 2. صفحة حصتي من الريع (`MySharePage.tsx`)

**المسار**: `/beneficiary/my-share`

**الدور**: عرض ملخص حصة المستفيد مع تفاصيل التوزيعات.

**البيانات المعروضة**:
- نسبة حصة المستفيد (%)
- المبلغ المستحق (حصته من `availableAmount`)
- ريع الوقف الإجمالي (`waqfRevenue`)
- حصة المستفيدين الإجمالية (`availableAmount`)

**المنطق الفريد**: هذه الصفحة تجلب **التوزيعات الفعلية** (`distributions`) الخاصة بالمستفيد:
```typescript
// جلب التوزيعات المرتبطة بالمستفيد الحالي
const { data: distributions } = useQuery({
  queryKey: ['my-distributions', currentBeneficiary?.id],
  queryFn: async () => {
    const { data } = await supabase
      .from('distributions')
      .select('*')
      .eq('beneficiary_id', currentBeneficiary.id);
    return data;
  },
});
```

**روابط ذكية**: توجه المستفيد لصفحة الإفصاح للتفاصيل المحاسبية العميقة.

### 3. صفحة التقارير المالية (`FinancialReportsPage.tsx`)

**المسار**: `/beneficiary/financial-reports`

**الدور**: تحليل بصري بالرسوم البيانية.

**البيانات المعروضة**:
- رسم بياني دائري لتوزيع الإيرادات حسب المصدر
- رسم بياني دائري لتوزيع المصروفات حسب النوع (**بدون ضريبة القيمة المضافة**)
- رسم بياني شريطي للبيانات الشهرية
- ملخص الأرقام الرئيسية

**المنطق الفريد**: تجميع البيانات الشهرية:
```typescript
// تجميع الدخل والمصروفات حسب الشهر للرسم البياني الشريطي
const monthlyData = useMemo(() => {
  const months = {};
  income.forEach(item => {
    const month = new Date(item.date).toLocaleDateString('ar-SA', { month: 'long' });
    months[month] = (months[month] || 0) + Number(item.amount);
  });
  return months;
}, [income]);
```

**استبعاد الضريبة**: تستخدم `expensesByTypeExcludingVat` لعرض المصروفات التشغيلية فقط (تستبعد بند "ضريبة القيمة المضافة المحصلة من الهيئة").

### 4. صفحة الحسابات الختامية (`AccountsViewPage.tsx`)

**المسار**: `/beneficiary/accounts`

**الدور**: عرض الحسابات الختامية مع تفاصيل العقود.

**البيانات المعروضة**:
- ملخص الحساب الختامي للسنة المالية المختارة
- إجمالي الدخل والمصروفات
- صافي ريع الوقف
- جدول العقود النشطة مع تفاصيل الإيجار

**المنطق الفريد**: تجلب بيانات العقود (فريدة لهذه الصفحة):
```typescript
const { data: contracts = [] } = useContracts();
// عرض العقود مع:
// - الإيجار السنوي (rent_amount)
// - الإيجار الشهري (rent_amount / 12)
// - حالة العقد (نشط/منتهي)
```

**عرض الإيجارات**: `rent_amount` يمثل دائماً القيمة السنوية. القيمة الشهرية تُحسب ديناميكياً حسب `payment_type`.

## هوك `useFinancialSummary` — المرجع الموحد

**الملف**: `src/hooks/useFinancialSummary.ts`

### المدخلات
```typescript
useFinancialSummary(fiscalYearId?: string, fiscalYearLabel?: string)
```
- `fiscalYearId`: لفلترة الدخل والمصروفات حسب السنة المالية
- `fiscalYearLabel`: لمطابقة الحساب الختامي المخزن (مثل "2024-2025")

### المخرجات
| القيمة | النوع | الوصف |
|--------|-------|-------|
| `income` | `Income[]` | سجلات الدخل الخام |
| `expenses` | `Expense[]` | سجلات المصروفات الخام |
| `beneficiaries` | `Beneficiary[]` | قائمة المستفيدين |
| `currentAccount` | `Account \| null` | الحساب الختامي المخزن (إن وجد) |
| `totalIncome` | `number` | إجمالي الإيرادات |
| `totalExpenses` | `number` | إجمالي المصروفات |
| `vatAmount` | `number` | الضريبة (يدوي) |
| `zakatAmount` | `number` | الزكاة (يدوي) |
| `waqfCorpusPrevious` | `number` | الرصيد المرحل |
| `waqfCorpusManual` | `number` | استقطاع رقبة الوقف |
| `distributionsAmount` | `number` | التوزيعات الفعلية |
| `grandTotal` | `number` | الإجمالي الشامل |
| `netAfterExpenses` | `number` | صافي بعد المصروفات |
| `netAfterVat` | `number` | صافي بعد الضريبة |
| `netAfterZakat` | `number` | صافي بعد الزكاة |
| `shareBase` | `number` | أساس حساب الحصص |
| `adminShare` | `number` | حصة الناظر |
| `waqifShare` | `number` | حصة الواقف |
| `waqfRevenue` | `number` | ريع الوقف |
| `availableAmount` | `number` | المبلغ القابل للتوزيع |
| `remainingBalance` | `number` | الرصيد المتبقي |
| `incomeBySource` | `Record<string, number>` | الدخل مجمّع حسب المصدر |
| `expensesByType` | `Record<string, number>` | المصروفات حسب النوع (شامل الضريبة) |
| `expensesByTypeExcludingVat` | `Record<string, number>` | المصروفات حسب النوع (بدون الضريبة) |
| `adminPct` | `number` | نسبة الناظر (من الإعدادات) |
| `waqifPct` | `number` | نسبة الواقف (من الإعدادات) |

### منطق اختيار الحساب
```
إذا وُجد fiscalYearLabel → ابحث عن حساب يطابق هذا التسمية
إذا لم يوجد + حساب واحد فقط → استخدمه
غير ذلك → null (حساب ديناميكي)
```

عند وجود حساب مخزن: تُقرأ القيم مباشرة منه (سجل مدقق).
عند عدم وجوده: تُحسب القيم ديناميكياً عبر `calculateFinancials()`.
